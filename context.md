# 项目背景

## 项目目标
做一个面向 Mac 的 AI 智能提词器 App，优先服务 Dewens 自己的 vibe coding、AI 产品演示、录屏讲解和远程展示。

它不是一个普通的滚动字幕工具，而是一个“AI 演示导演”：帮助用户把零散材料整理成自然口播稿，并在演示时用低打扰、精致、带仪式感的界面辅助表达。

## 适用场景
- Vibe coding 过程讲解：边展示产品/代码/页面，边讲清楚思路和关键步骤。
- AI App demo 录制：把功能亮点、操作路径、背景判断组织成 3-5 分钟自然口播。
- 产品方案表达：把 PRD、想法、客户反馈、技术实现整理成适合对外讲解的稿子。
- 远程会议/直播演示：以悬浮提词层帮助表达，不抢占主屏幕注意力。

## 当前约束
- 时间：先完成需求定义、参考分析和 MVP 原型方向，再进入实现。
- 资源：已有一个抖音 AI 电台 vibe coding 参考视频，作为产品气质和 UI 设计语言参考。
- 技术：目标形态为 Mac 原生 App；第一版优先验证材料转口播稿和提词体验，不先追求复杂实时语音跟随。
- 协作：先对齐产品核心需求与 UI 方向，再逐步沉淀 PRD、原型和实现任务。

## 验收标准
当前需求定义阶段完成的标准：
- 明确产品定位、核心用户、MVP 能力、暂不做范围。
- 完成参考视频拆解，提炼可迁移的 UI/交互/产品气质原则。
- 形成下一步可进入原型设计或技术选型的稳定文档。

第一版产品验收标准：
- Dewens 可以用它把真实 vibe coding / AI App demo 材料生成 3-5 分钟可讲的口播稿。
- Mac 提词层在录屏/演示时好读、不挡关键内容、有沉浸感。
- 界面气质接近“暗色精致工作台 + AI 演示导演”，而不是普通文本工具。

## 重要背景
- 产品定位来自 2026-05-02 的需求讨论：第一版服务 Dewens 自己，而不是先做通用大众提词器。
- 设计语言参考 `artifacts/raw/reference-videos/douyin-mmguo-ai-radio-vibe-coding-reference-2026-05-02.mp4`，重点学习其暗色、微光、状态感、AI 伙伴身份和演示仪式感。
- 参考方向不是复刻 Claudio / AI 电台概念，而是吸收“把工具做成有身份、有状态、有情绪的创作空间”的设计思想。
