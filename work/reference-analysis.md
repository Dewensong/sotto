## 参考价值
这个视频最值得借鉴的不是“电台”题材本身，而是它把一个 AI 工具包装成了一个有身份、有情绪、有状态的产品体验。

Claudio 看起来不是一个普通播放器，也不是一个普通聊天框。它像一个可以陪伴用户的 AI DJ，有自己的直播状态、时间、声音、播报、音乐选择和对话入口。这个思路可以迁移到我们的 AI 提词器：它也不应该只是滚动字幕，而应该像一个“演示导演”，帮用户进入表达状态。

## 产品思想拆解
- **工具有身份**：Claudio 使用名字、头像、`Speaking...`、`On Air` 等状态，让用户感到系统正在主动陪伴，而不是静态工具。
- **体验有仪式感**：从终端启动、连接服务、加载偏好，到进入播放器界面，整个过程像“开播”而不是“打开网页”。
- **内容有场景感**：它根据时间、日程、天气、收藏夹和音乐风格生成播报，不只是展示数据。
- **界面围绕注意力设计**：当前播放/当前播报最突出，其他内容弱化，符合视频里“attention is a spotlight”的表达。
- **技术结构被产品化表达**：服务端、API、播放器、模型、音乐源被讲成一个可理解的产品系统，而不是堆技术名词。

## UI 语言拆解
- **暗色基底**：整体以黑色和深灰为主，营造夜间、专注、沉浸的创作氛围。
- **局部荧光**：绿色、青色、紫色和粉色只作为状态点、边框、光效或强调，不大面积铺开。
- **像素/终端质感**：像素字体、点阵、命令行窗口、低亮度网格，让产品有“被创造出来”的 vibe coding 气质。
- **卡片与控制台并存**：对外体验是圆角播放器卡片，对内构建过程是终端/控制台/结构图。
- **前后层次明显**：当前内容高亮，历史和未来内容降低透明度，形成清晰阅读焦点。

## 可迁移到提词器的方向
- 把产品人格定义为“AI 演示导演”，而不是普通文本助手。
- 将核心状态设计出来：`Preparing`、`Rehearsing`、`Live Prompting`、`On Air`、`Paused`。
- 提词区域采用“当前句聚光灯”：当前句高亮，上一句/下一句弱化，帮助用户自然接续。
- 主界面可以像暗色工作台：左侧材料输入，中间口播稿，右侧演示状态/段落结构/节奏提示。
- 提词浮窗应更像“专注层”：少按钮、低干扰、保留进度、节奏和下一句提示。
- AI 生成稿件时要有“导演感”：不是只返回文本，而是帮助用户组织开场、转场、重点、收束。

## 不应直接照搬
- 不直接复制 Claudio 名称、视觉品牌、电台设定、播放器结构。
- 不把产品做成强复古像素皮肤；我们的方向是暗色精致工作台，复古/终端感只作为局部气质。
- 不让视觉氛围压过可读性。提词器的第一原则仍然是好读、稳定、不打断表达。
- 不在第一版追求复杂实时语音跟随或全自动导演，先保证“材料转口播稿 + 提词体验”闭环。

## 对本项目的设计原则
1. **先满足核心表达需求，再做风格**：UI 好看必须服务“讲得清楚、讲得自然”。
2. **仪式感来自状态流转**：准备稿子、排练、上场、暂停、收尾都应该有清晰状态。
3. **当前内容永远最重要**：所有视觉层级都要让用户快速看到此刻该讲什么。
4. **AI 像导演，不像客服**：文案、状态、按钮和改写能力都要体现它在帮用户组织表达。
5. **Mac 演示场景优先**：悬浮窗、快捷键、透明度、屏幕空间占用都要围绕真实录屏/会议使用。
