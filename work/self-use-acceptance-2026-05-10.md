# Sotto 自用验收计划（2026-05-10）

## 目的
这轮验收不把 Sotto 当作功能清单检查，而是把它当作 Dewens 下一条 vibe coding / AI 产品演示视频的真实提词工具来试。

目标是回答三个问题：
- 它是否能支撑 3-5 分钟真实口播，而不是只在短样本文案里可用？
- 它是否适合出现在 Dewens 的录屏工作流里，包括窗口位置、遮挡、屏幕共享隐藏和现场调节？
- 它是否已经足够成为一个可公开展示的 vibe coding 成果？

## 验收材料
- 一段真实 3-5 分钟口播稿，建议来自最近一次真实 AI 产品 / vibe coding demo。
- 一个真实录屏场景：主屏展示代码、网页或产品界面，Sotto 提词窗口悬浮在真实画面上。
- 两种播放模式各试一次：定时模式、跟声模式。
- 一次开启屏幕共享隐藏的录屏测试，确认录屏画面是否符合预期。

## 验收流程
1. 从首页粘贴真实稿件，点击准备上场。
2. 检查稿件是否自动进入稿件库，并能从“管理全部”再次打开。
3. 在编辑页完成至少 5 次真实编辑：改一句、拆一句、合一句、加长停顿、设强调。
4. 打开正式提词窗口，调整位置、字号、透明度、宽度和亮度。
5. 用定时模式完整讲一遍，记录节奏偏快 / 偏慢位置。
6. 用跟声模式完整讲一遍，记录静音暂停、开口推进、阈值误触发或不跟声的问题。
7. 在真实录屏工具里验证屏幕共享隐藏：确认自己可见，录屏 / 共享画面按预期显示或隐藏。
8. 试一次长稿滚动和点击跳句，确认临场纠错是否顺手。

## 观察维度

### 口播可用性
- 当前句是否足够醒目。
- 上一句 / 下一句层级是否帮助接续，而不是干扰。
- token 光标是否帮助读稿，还是造成焦虑。
- 长句是否换行自然，是否需要手动缩字号。

### 现场操控
- TUNE 抽屉是否容易找到。
- 播放 / 暂停是否足够灵敏。
- 拖动窗口和拖动滑杆是否冲突。
- 四角 / 边缘 resize 是否可发现、可控。
- 快捷键是否值得作为主操作强化。

### 录屏安全
- 提词窗口是否遮挡关键代码、网页、产品界面。
- 屏幕共享隐藏是否符合预期。
- 主编辑窗口进入提词后隐藏是否减少噪音。
- 关闭提词后是否能顺利回到编辑状态。

### 稳定性
- 3-5 分钟讲完是否出现卡顿、错位、跳句或麦克风异常。
- 长稿是否让编辑页或正式提词窗口明显变慢。
- 麦克风授权、拒绝授权、无输入设备时是否有可理解的反馈。

## 验收通过标准
- Dewens 能用 Sotto 完成一条 3-5 分钟真实 demo 口播，不需要中途放弃工具。
- 正式提词窗口在录屏中可读、不抢画面、不频繁打断操作。
- 跟声模式即使不完美，也不会比定时模式更危险；若不稳定，公开版 README 中应明确其仍是实验能力。
- 屏幕共享隐藏的实际表现被验证并记录，公开描述不夸大。
- 公开仓库前至少有 3 张可用截图或 1 段短演示素材。
