## 交互含义

这张图将 MVP 0.1 的切分交互表达得比较具体：

- 用户当前正在编辑某一句。
- 系统把该句拆成若干短语。
- 用户可以在短语之间点击插入 / 取消切分点。
- 用户可以为切分点或当前句设置停顿。
- 用户可以设置当前句是否强调。
- 所有调整会影响底部当前句预览。

## 设计原则

- 切分面板要像「节奏调整台」，不是复杂参数表。
- 当前句应该以足够大的字号呈现，方便用户确认语义。
- 切分点要有明确的触点和发光反馈。
- 停顿和强调设置要直观，优先使用分段控件。
- 当前句预览要继承正式提词窗口的视觉语言。
- 面板整体要保持暗色、点阵、微光和演出后台气质。

## 后续使用方式

后续生成 UI、动效或实现切分面板时，应优先参考这张图的：

- 当前选中句子大号展示。
- 横向短语切分卡片。
- 发光切分点。
- 分段式停顿 / 强调控制。
- 当前句耗时展示。
- 底部当前句提词预览。

如需调整，优先保留「大句子确认 + 可点击切分点 + 即时预览」这条主线。
