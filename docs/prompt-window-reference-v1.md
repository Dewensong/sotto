## 采纳结论

当前提词窗口方向被采纳为 Sotto MVP 0.1 的正式提词窗口参考。

## 画面结构

提词窗口采用横向悬浮卡片结构，主要分为：

1. 左侧状态栏。
2. 主提词阅读区。
3. 底部节奏 / 波形线。
4. 底部控制栏。

## 关键视觉元素

- 暗色半透明横向卡片。
- 圆角发光边框。
- 点阵背景。
- 像素 / 点阵字体。
- 左侧 `SOTTO PROMPT` 状态栏。
- 绿色状态点。
- `REC` 和录制时长。
- 当前进度，例如 `12 / 48`。
- 主阅读区中当前句最亮、最大。
- 上一句在上方弱化。
- 下方多句作为未来句预告。
- 中部横向光束，强化聚光感。
- 底部细节奏线 / 波形线。
- 控制按钮：上一句、暂停、下一句、速度 -、速度值、速度 +、退出。

## 文案与显示关系

画面展示的提词层级符合 MVP 0.1 的「聚光流」方向：

- 上一句弱化保留。
- 当前句最突出。
- 下一句及后续句清楚可读，但弱于当前句。
- 当前短语 / 当前句通过更亮的文字和光感聚焦。

示例文案：

- 上一句：`今天我们不急着说服谁。`
- 当前句：`我们只是在上场前，把真正想说的话，慢慢说清楚。`
- 未来句：`先让呼吸稳定下来，再让句子往前走。`

## 设计原则

- 正式提词窗口是 Sotto 的核心体验，优先级高于其他页面。
- 它应该像一个安静的舞台提词屏，不像普通网页弹窗。
- 当前内容要有被聚光照亮的感觉。
- 控制区要清楚但克制，不能抢提词内容的注意力。
- 视觉可以有强烈气质，但文字可读性永远优先。

## 后续使用方式

后续生成 UI、动效或实现提词窗口时，应优先参考这张图的：

- 横向悬浮卡片比例。
- 左侧状态栏。
- 大号当前句。
- 点阵背景。
- 中部光束。
- 底部节奏线。
- 克制的播放控制栏。

如需调整，优先保留「暗色悬浮卡片 + 聚光当前句 + 安静控制台」这条主线。
