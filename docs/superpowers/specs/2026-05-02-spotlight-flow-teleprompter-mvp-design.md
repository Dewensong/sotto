# 聚光流提词器 MVP 0.1 设计规格

## 1. 定位

MVP 0.1 是一个 **Mac 本机聚光流提词器**。它先不做 AI 生成稿件，也不做完整内容策划流程，只验证一个最核心的问题：

> 用户已经有一份稿件时，产品能否把它变成一种舒服、安心、艺术化的提词体验，帮助用户更稳定地录制口播视频。

第一版服务 Dewens 自己的抖音 / 小红书内容录制，重点解决「看着稿子也会口误、反复重录、录制前不安心」的问题。

## 2. MVP 0.1 范围

### 必须包含

- 粘贴已有稿件。
- 自动切分句子和短语。
- 支持手动调整短语切分。
- 简单速度滑杆。
- 提词预览区。
- Mac 本机提词窗口。
- 中等宽度字幕卡片，默认浮在屏幕中央偏上方。
- 聚光流显示：上 1 句 + 当前 1 句 + 下 3 句。
- 当前短语用柔光扫读，不使用传统 KTV 色块填字。
- 按预设速度自动推进。
- 鼠标 / 触控板控制暂停、继续、速度微调。
- 默认视觉主题：暗色半透明卡片 + 柔光。

### 明确不包含

- AI 生成稿件。
- 创作蓝图。
- 素材校准。
- Notion 自动接入。
- 外部视频字幕提取。
- GitHub 项目分析。
- 要点模式。
- Side Cue 素材提示。
- 风格模板。
- 可说性检查。
- 语音识别校准。
- 「自由发挥中」状态。
- Screen Studio 录制隐藏 / Ghost Mode。
- iPad / 手机第二屏提词。

## 3. 核心用户场景

用户已有一份口播稿，可能来自 Notion、AI 对话、已有文稿或手动整理。用户打开 App 后：

1. 把稿件粘贴进来。
2. 系统自动切分成适合口播的句子和短语。
3. 用户在文本编辑器里检查稿件，并在切分面板里微调短语边界。
4. 用户用速度滑杆调整节奏，在提词预览区确认读起来是否舒服。
5. 用户打开 Mac 本机提词窗口。
6. 提词窗口用聚光流方式推进稿件，用户按节奏录制。

MVP 0.1 允许提词窗口被 Screen Studio 全屏录制捕获。此版本目标是验证提词体验，不保证正式成片可直接使用。

## 4. 主界面：演出后台入口

第一屏不做完整项目系统，先提供一个安静、有仪式感的材料入口。

推荐文案：

```text
今天想让哪段想法上场？
把已经写好的稿子放进来，我会帮你整理成适合提词的节奏。
```

主操作：

- 粘贴稿件。
- 创建提词稿。

界面气质：

- 像录制前的演出后台，而不是任务管理器。
- 暗色、克制、留白充足。
- 不展示大量功能入口。

## 5. 稿件编辑与切分

### 文本编辑器

稿件编辑区更像文本编辑器，而不是字幕时间轴。

- 展示连续稿件和自然段落。
- 支持直接修改文字。
- 轻量显示句子和短语切分标记。
- 当前预览句可以轻微高亮。
- 不在正文里放大量按钮。

### 自动切分

自动切分默认取「适中」策略。

- 不按每个词切，避免机器感。
- 不整句一块，避免高亮太粗。
- 按自然口播停顿切成短语。
- 长句自动拆成 2-4 个短语。
- 短句保持完整。
- 优先参考逗号、顿号、转折词、强调词和语义边界。

示例：

```text
最近我看到一个｜很有意思的 AI 电台项目。
它不是普通播放器，｜更像一个｜会根据你状态说话的 AI DJ。
```

### 切分面板

手动调整放在侧边切分面板中，主稿件保持干净。

当用户选中一句话时，面板显示：

- 当前句文本。
- 短语切分结果。
- 点击插入 / 取消切分点。
- 停顿设置。
- 强调设置（可作为轻量状态，不进入复杂样式系统）。
- 当前句预计耗时。
- 预览当前句。

第一版短语调整只支持点击插入 / 取消切分点，不做拖拽。

## 6. 节奏设置与预览

### 速度滑杆

第一版使用简单速度滑杆。

- UI 文案偏感性，不暴露复杂参数。
- 可表达为：舒缓、标准、紧凑。
- 调整速度时，同步更新预计时长。
- 提词现场也允许用按钮轻量微调速度。

### 提词预览区

编辑页必须有提词预览区，用来确认聚光流实际观感。

预览区显示：

- 上 1 句。
- 当前 1 句。
- 下 3 句。
- 当前短语柔光扫读。
- 当前速度设置下的推进效果。

预览区的目标是帮助用户在正式进入提词窗口前判断：

- 句子切分是否顺口。
- 高亮节奏是否舒服。
- 未来句是否足够清楚。
- 整体视觉是否安心。

## 7. 提词窗口

### 默认形态

- Mac 本机浮窗。
- 中等宽度字幕卡片。
- 默认位置：屏幕中央偏上方。
- 暗色半透明背景。
- 轻微柔光和呼吸感。
- 可拖动位置。
- 可调整窗口宽度、字号和透明度。

### 聚光流结构

一屏显示 5 句：

```text
上一句：弱化保留
当前句：最突出
下一句：清楚可读
下下句：清楚可读，稍弱
下下下句：预告，可读但更轻
```

当前句不只整句高亮，而是按短语逐步高亮。

### 聚光高亮

高亮方式采用「柔光扫读 + 文字微亮」。

- 当前短语文字更亮、更清晰。
- 背后有一层很淡的柔光或半透明底色扫过。
- 不做传统 KTV 色块填字。
- 不做强闪烁或高饱和动画。
- 扫过后文字回到当前句正常亮度。

视觉目标：

> 不是「唱到这里了」，而是「这句话正在被照亮」。

### 推进方式

MVP 0.1 只做预设速度推进，不做语音校准。

- 高亮按速度滑杆推进。
- 用户可以暂停 / 继续。
- 用户可以手动上一句 / 下一句。
- 用户可以微调快慢。

「停顿时呼吸惯性」属于 0.2 需求，MVP 0.1 暂不实现。

### 控制方式

第一版以鼠标 / 触控板点击按钮为主。

提词窗口保留轻量控制：

- 暂停 / 继续。
- 上一句 / 下一句。
- 速度稍慢 / 稍快。
- 退出提词。

按钮平时弱化，靠近或悬停时更清楚。按钮要足够大，避免录制时点不到。

## 8. 视觉原则

MVP 0.1 默认只做一个主题：

**暗色半透明卡片 + 柔光。**

设计原则：

- 像演出后台，不像办公表格。
- 有仪式感，但不戏剧化。
- 有艺术感，但不影响可读性。
- 当前内容最清楚，未来内容也足够清楚。
- UI 服务安心感，不制造录制压力。

## 9. 后续版本边界

### 0.2：智能跟随

- 语音识别校准。
- 停顿时缓慢继续一点点，再温柔停住。
- 自由发挥中状态。
- 偏离稿件时安静等待。

### 0.3：录制安全

- 验证 Screen Studio 全屏录制下的隐藏提词层。
- 如 Mac 隐形层不稳定，提供 iPad / 手机第二屏安全录制模式。

### 0.4+：内容准备能力

- 素材校准。
- 创作蓝图。
- 完整稿件 + 要点大纲生成。
- 风格模板：夜间讲解、快切推进、深水观点。
- Side Cue 段落级素材提示。
- Notion / 外部链接 / GitHub 材料接入。

## 10. 规格自检

- 没有把 AI 生成稿件纳入 MVP 0.1。
- 没有把语音识别、Ghost Mode、第二屏提词纳入 MVP 0.1。
- 聚光流显示规则明确：上 1 句 + 当前 1 句 + 下 3 句。
- 编辑器形态明确：主区是文本编辑器，侧边是切分面板。
- 手动切分方式明确：点击插入 / 取消切分点。
- 视觉主题明确：暗色半透明卡片 + 柔光。
- 验证目标明确：先验证提词体验，不承诺正式录制隐藏。
