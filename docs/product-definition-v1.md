# AI 智能提词器产品定义 V1

## 一句话定位
面向 vibe coding 和 AI 产品演示的 Mac 原生 AI 提词器，把零散材料变成自然口播稿，并在演示时用精致、低打扰的提词层辅助表达。

## 核心用户
第一版先服务 Dewens 自己：
- AI 产品经理
- 经常做 vibe coding、产品 demo、AI App 讲解
- 需要把想法、代码、功能、方案快速组织成可讲述内容
- 关注产品气质、界面审美和表达质量

## 核心需求
用户真正需要的不是“文字滚动”，而是：
- 快速把材料变成适合说出口的讲解稿。
- 在演示时知道当前该讲什么、下一步怎么转。
- 不被复杂编辑器或笨重 UI 打断表达状态。
- 让录屏/会议/demo 过程更稳定、更自然、更有掌控感。

## 产品人格
暂定为“AI 演示导演”。

它应该像一个在后台帮用户控场的搭档：
- 帮用户组织讲解结构。
- 提醒开场、转场、重点和结尾。
- 在提词时保持安静、克制、专注。
- 用状态感让用户知道自己正在准备、排练还是正式演示。

## MVP 功能
- 材料输入：支持粘贴想法、产品说明、代码变更、演示步骤。
- 口播生成：生成按段落组织的自然口播稿。
- 改写指令：支持更自然、更短、更像我、更适合开场、更适合结尾。
- 稿件结构：默认分为开场、背景、演示步骤、关键亮点、结尾行动点。
- 主工作台：暗色精致界面，展示材料、生成稿、段落结构和演示状态。
- 提词浮窗：Mac 悬浮窗，支持置顶、透明度、字号、宽度、位置和快捷键。
- 演示模式：突出当前句，弱化前后句，显示下一句提示和整体进度。

## 暂不做
- 不做完整视频录制/剪辑。
- 不做多人协作。
- 不做复杂实时读屏和全自动演示导演。
- 不把语音跟随作为第一版硬依赖。
- 不直接复刻 Claudio / AI 电台的品牌和具体界面。

## UI / 设计语言
设计方向：暗色精致工作台。

参考 Claudio 视频的气质：
- 暗色、微光、沉浸。
- 有明确状态感和启动仪式。
- 局部终端/像素/vibe coding 气质。
- 当前内容像被聚光灯照亮。
- AI 不是工具按钮，而是有身份的陪伴者。

落到本产品：
- 主界面像演示控制台。
- 提词浮窗像专注阅读层。
- 状态包括 `Preparing`、`Rehearsing`、`Live Prompting`、`On Air`、`Paused`。
- 视觉强调用低饱和荧光色，避免大面积花哨装饰。
- 文案要短、稳、有导演感，不使用教程式说明堆满界面。

## 第一版验收
- Dewens 可以用真实 vibe coding / AI App demo 材料生成 3-5 分钟口播稿。
- 生成稿能自然开场、解释背景、串起操作、突出亮点、完成收束。
- 提词界面在 Mac 演示/录屏场景中可读、低干扰、有沉浸感。
- UI 气质接近参考视频的暗色、精致、有仪式感方向，但仍保持专业生产力工具属性。

## 下一步
1. 画第一版信息架构：材料输入、稿件编辑、演示模式、设置。
2. 设计 2-3 个关键界面草图：主工作台、提词浮窗、AI 改写面板。
3. 确认技术路线：SwiftUI 原生 App、Tauri，或先用 Web 原型验证交互。
4. 选一段真实 demo 材料，作为 MVP 测试样本。
