# MVP 0.1 页面描述参考

> 重要说明：这份文档只是阶段性参考，用来帮助后续找视觉风格、生成 UI 概念图、和其他 AI 沟通页面需求。它不是最终设计稿，也不是固定方案。Dewens 后续可以随时调整页面数量、页面结构、命名、交互和视觉方向。

## 页面总览

MVP 0.1 暂时包含以下页面和状态：

1. 首页 / 演出后台入口
2. 稿件编辑页
3. 切分面板
4. 提词预览区
5. 正式提词窗口
6. 提词窗口设置浮层
7. 自动切分处理中状态

这些页面共同服务一个核心流程：

```text
粘贴已有稿件 -> 自动切分 -> 手动微调 -> 预览聚光流 -> 打开 Mac 本机提词窗口
```

## 1. 首页 / 演出后台入口

### 页面目的

这是用户打开 App 后看到的第一屏。它的目标不是展示复杂功能，而是让用户快速进入「准备一段表达」的状态。

这一页应该像录制前的演出后台：安静、有仪式感、让人安心。它不应该像任务管理器、文档库、数据库表格或普通文本编辑器。

### 页面内容

- 产品标识或 App 名称。
- 设置 / 偏好入口。
- 一句具有氛围感的主标题，例如「今天想让哪段想法上场？」。
- 一段简短说明，告诉用户可以粘贴已有稿件。
- 主材料输入区，用来粘贴口播稿、Notion 文稿、AI 对话整理稿等大段文本。
- 主按钮，例如「创建提词稿」「准备上场」。
- 最近稿件 / 草稿列表，作为轻量继续入口。

### 设计重点

- 主视觉焦点应该是材料输入区。
- 最近稿件不能抢主入口的注意力。
- 文案要有氛围，但功能含义必须清楚。
- 整体感觉应该是「把想法放进后台，准备上场」，不是「开始一个办公任务」。

## 2. 稿件编辑页

### 页面目的

用户粘贴稿件后进入这一页。这里用于阅读、修改稿件，并查看系统自动切分后的结果。

页面主区应该像一个干净的文本编辑器，而不是字幕软件或剪辑软件。用户需要感觉自己还在编辑一篇稿子，而不是被迫进入复杂的时间轴工作流。

### 页面内容

- 顶部返回入口。
- 稿件标题或可编辑名称。
- 打开正式提词窗口的入口。
- 主文本编辑区。
- 自动切分后的句子和短语边界。
- 当前选中句子的轻量高亮。
- 底部或侧边的节奏设置。
- 嵌入式提词预览入口或预览区域。

### 设计重点

- 文本编辑区是主角。
- 切分标记要轻，不要让正文看起来支离破碎。
- 用户应该可以直接修改文字。
- 页面要保持安静、可读、低压力。

## 3. 切分面板

### 页面目的

切分面板用于调整某一句话的短语切分、停顿和强调。它是稿件编辑页的辅助区域，不是独立主页面。

用户点击或选中某一句后，切分面板展示这句话的详细控制。

### 页面内容

- 当前选中句子的完整文本。
- 当前句子的短语切分结果。
- 用于点击插入 / 取消切分点的交互区域。
- 停顿设置：无、短停顿、长停顿。
- 强调设置：普通、强调。
- 当前句预计耗时。
- 预览当前句的按钮或小区域。

### 设计重点

- 切分面板应该像「导演备注」或「节奏卡」，不是工程参数面板。
- 第一版只需要点击插入 / 取消切分点，不需要拖拽。
- 面板信息要清楚，但不要显得复杂。
- 它的视觉权重应该低于主稿件文本。

## 4. 提词预览区

### 页面目的

提词预览区用于在正式录制前预览聚光流效果。用户可以确认句子切分、短语高亮、速度和文字层级是否舒服。

它可以嵌入在稿件编辑页中，也可以作为一个独立的预览卡片出现。

### 页面内容

- 上一句。
- 当前句。
- 下 3 句。
- 当前短语的柔光扫读效果。
- 暂停 / 继续控制。
- 当前速度状态。
- 当前句序号。
- 预计总时长。

### 设计重点

- 预览区只服务「看提词效果」，不承担复杂编辑。
- 当前句必须最突出。
- 下一句、下下句、下下下句要清楚可读。
- 上一句要保留上下文，但明显弱化。
- 柔光扫读要克制、艺术化，不要像传统 KTV 填色。

## 5. 正式提词窗口

### 页面目的

这是录制时真正使用的 Mac 本机浮窗。它是 MVP 0.1 最重要的体验核心。

用户会看着它读稿，所以它必须稳定、好读、安心、低干扰。

### 页面内容

- 中等宽度的字幕卡片。
- 默认放在屏幕中央偏上方。
- 上一句。
- 当前句。
- 下 3 句。
- 当前短语柔光高亮。
- 暂停 / 继续按钮。
- 上一句 / 下一句按钮。
- 速度微调按钮或滑杆。
- 退出提词按钮。

### 设计重点

- 不能全屏。
- 不能像普通网页弹窗。
- 它应该像一个安静的提词卡片，悬浮在录制前方。
- 背景建议是暗色半透明，并有轻微柔光。
- 控制按钮平时弱化，鼠标靠近时更清楚。
- 文字可读性优先于所有装饰。

## 6. 提词窗口设置浮层

### 页面目的

用于调整提词窗口的显示效果。它是轻量设置，不是完整设置页。

### 页面内容

- 字号设置。
- 窗口宽度设置。
- 透明度设置。
- 亮度设置。
- 位置预设：中央偏上、靠近摄像头、自定义拖动。
- 当前主题说明：暗色半透明 + 柔光。
- 完成按钮。

### 设计重点

- 只放影响提词可读性的设置。
- 不做复杂主题市场或高级配置。
- 设置浮层应该轻，不要破坏提词状态。
- MVP 0.1 只需要一个默认主题。

## 7. 自动切分处理中状态

### 页面目的

用户粘贴稿件后，系统需要短暂处理文本，识别段落、句子和短语。这个状态用于承接等待过程。

### 页面内容

- 一句安静的状态文案，例如「正在整理这段稿子的提词节奏」。
- 当前处理步骤：识别段落、切分句子、生成短语高亮节奏。
- 轻微呼吸光或状态点。

### 设计重点

- 等待状态要温和，不要像系统任务进度条。
- 不使用强警告色。
- 不制造焦虑。
- 处理完成后自然进入稿件编辑页。
