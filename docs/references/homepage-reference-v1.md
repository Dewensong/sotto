# Sotto 首页参考 V1

> 说明：这是 2026-05-02 由 Dewens 借助 GPT 网页端 DALL-E 3 反复调整后采纳的首页方向。它作为 Sotto 首页视觉与气质参考，不代表最终 UI 已锁死，后续仍可继续调整。

## 采纳结论

当前首页方向被采纳为 Sotto MVP 0.1 的首页参考。

产品名：

- **Sotto**

产品气质句：

- `a quiet backstage for speaking well`

首页核心文案：

- `今天想让哪段想法上场？`
- `把已经写好的稿子放进来，我会帮你整理成适合口播的提词节奏。`

主按钮：

- `ON STAGE`
- `准备上场`

## 画面结构

首页采用纵向 Mac 桌面 App 画面结构，整体分为：

1. 顶部状态区
2. 品牌与标识区
3. 主标题与说明区
4. 主材料入口卡片
5. 最近稿件列表
6. 底部气质标语

## 关键视觉元素

- 暗色背景。
- 点阵网格。
- 顶部舞台光束。
- 像素 / 点阵风格产品标识。
- 左上角 Mac 窗口三色圆点。
- 右上角 `READY.` 状态点和设置图标。
- 大面积半透明材料入口卡片。
- 中央文档图标。
- 功能提示：段落识别、短语切分、停顿建议、聚光预览。
- 类似音频波形 / 节奏线的横向装饰。
- 高仪式感主 CTA：`ON STAGE / 准备上场`。
- 最近稿件列表使用卡片行展示，并带状态点。

## 首页内容模块

### 品牌区

用于建立产品身份和系列感。

- 图形标识：点阵风格，像提词屏 / 舞台屏 / 控制台符号。
- 产品名：`Sotto`。
- 副标题：`a quiet backstage for speaking well`。

### 状态区

用于表达产品处于安静待命状态。

- `READY.`
- 绿色状态点。
- 设置入口。

### 主入口区

这是首页最重要的区域。

用途：

- 让用户粘贴已有口播稿、Notion 文稿或 AI 对话整理稿。
- 强调支持大段文本粘贴。
- 告诉用户系统会自动切分句子与节奏。

视觉要求：

- 主入口卡片要有舞台后台感。
- 不像普通输入框。
- 中央文档图标要有轻微发光。
- 文案要清楚，但不要工具化。

### 能力提示区

显示 MVP 0.1 的核心能力：

- 段落识别。
- 短语切分。
- 停顿建议。
- 聚光预览。

这些能力提示只作为辅助说明，视觉权重要低于主入口和主 CTA。

### 最近稿件区

展示最近稿件，但不能抢主入口注意力。

示例稿件：

- 产品发布会开场稿：待持续。
- 创作者成长分享：已切分。
- 品牌故事叙述稿：润词中。

最近稿件采用暗色列表卡片，带字母图标、最后编辑时间、状态点和更多菜单。

### 底部标语

- `专注表达，聚光成句。`

## 设计原则

- 首页要像「演出后台」，不是普通项目管理器。
- 主要行为是「准备上场」，不是「创建文件」。
- 点阵、光束、暗色和微光可以成为 Sotto 的核心视觉语言。
- UI 可以有艺术感，但材料入口和主按钮必须清晰可用。
- 首页要让用户感到安心、专注、有一点仪式感。

## 后续使用方式

后续生成 UI 或实现首页时，应优先参考这版首页的：

- 品牌气质。
- 暗色点阵语言。
- 舞台光感。
- 主 CTA 形式。
- 最近稿件状态表达。
- 「演出后台」隐喻。

如需调整，优先保留「Sotto / backstage / on stage / spotlight」这条气质主线。
