## 画面结构

稿件编辑页采用纵向 Mac 桌面 App 结构，主要分为：

1. 顶部稿件状态栏。
2. 左侧主稿件编辑区。
3. 右侧切分 / 状态 / 节奏面板。
4. 底部提词预览区。
5. 最底部工具栏。

- 最底部工具栏包含：撤销、重做、导入文本、导出稿件、更多。

## 页面内容关系

这张图很好地表达了 MVP 0.1 的中间工作区：

- 用户不是在复杂字幕软件里工作，而是在一个安静的稿件准备台里调整提词节奏。
- 主稿件仍然是文本编辑器，保持阅读和修改的自然感。
- 切分、状态、节奏作为右侧辅助面板，不抢主稿件注意力。
- 提词预览区位于底部，帮助用户随时确认「正式提词时会怎么读」。

## 设计原则

- 稿件编辑区是主角，右侧面板是辅助。
- 当前段落高亮要清楚，但不能像报错或强提醒。
- 切分标记可以明显一些，但不要破坏正文阅读感。
- 提词预览区应该继承正式提词窗口的暗色、微光、横向控制台质感。
- 工具栏要克制，避免把页面做成普通办公软件。

## 后续使用方式

后续生成 UI、动效或实现稿件编辑页时，应优先参考这张图的：

- 左侧稿件编辑 + 右侧辅助面板 + 底部提词预览布局。
- 当前段落发光选中态。
- 顶部稿件标题和自动保存状态。
- 右上角打开提词窗口入口。
- 底部嵌入式提词预览。
- 暗色点阵后台气质。

如需调整，优先保留「稿件是主角，切分和预览围绕稿件服务」这条主线。
