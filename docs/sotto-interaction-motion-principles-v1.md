# Sotto 交互与动效原则 V1

> 说明：这份文档基于参考视频 `artifacts/raw/reference-videos/douyin-mmguo-ai-radio-vibe-coding-reference-2026-05-02.mp4` 的重新观察，以及 Sotto 已采纳的 5 张 UI 参考图整理而来。它用于指导后续原型、视觉生成和实现，不是最终动效规格。

## 1. 核心判断

Sotto 的动效目标不是「让界面动起来」，而是让用户感觉：

> 这个空间正在安静地呼吸，系统在后台陪我准备表达。

参考视频里最值得迁移的不是炫技转场，而是低速、持续、可感知但不打扰的小生命感：

- 播放器的波形一直在动，让系统像正在播报。
- `Speaking...` / `On Air` 这类状态点提供持续陪伴感。
- 当前文字被绿色轻轻标记，历史内容自然退后。
- 背景光雾和暗色空间让界面像一个被点亮的舞台。
- 页面切换多是淡入、推进、聚焦，而不是夸张弹跳。

迁移到 Sotto 时，动效应服务 3 件事：

1. **安心**：用户知道系统还在工作，没有卡住。
2. **聚焦**：用户知道此刻该看哪里、该讲哪一句。
3. **节奏**：用户能感到稿件正在被整理、预演和推进。

## 2. 动效气质

Sotto 的动效应该像「静默呼吸」：

- 慢。
- 稳。
- 柔。
- 有方向。
- 有状态。
- 低干扰。

它不应该像游戏、短视频模板、营销页或加载特效合集。

推荐关键词：

- 呼吸。
- 聚光。
- 扫读。
- 浮现。
- 退后。
- 微亮。
- 轻触。
- 安静等待。

避免关键词：

- 弹跳。
- 震动。
- 炫彩。
- 爆闪。
- 翻转。
- 大幅滑入。
- 快速循环。
- 强烈粒子喷发。

## 3. 运动层级

Sotto 的界面可以分为 4 个运动层级，越靠后越慢、越弱。

### 3.1 背景层

背景层负责建立「有生命的空间」，但不能抢内容。

包括：

- 暗色点阵。
- 轻微颗粒。
- 顶部舞台光。
- 背景光尘。
- 底部声波 / 节奏线。

原则：

- 背景始终在非常轻地变化。
- 变化周期要长，适合 6-12 秒一轮。
- 不要出现明显方向性强的循环纹理，避免用户觉得界面在晃。
- 点阵生命感来自亮度呼吸、局部随机闪烁和极慢位移，而不是大面积流动。

### 3.2 状态层

状态层负责告诉用户「Sotto 在场」。

包括：

- `READY.`
- `PREPARING`
- `在线`
- `SOTTO PROMPT`
- `REC`
- 当前步骤节点。

原则：

- 状态点可以持续微光呼吸。
- 呼吸幅度要小，亮度变化可被看见，但不能像警报。
- 处理中状态要像准备上场，不像系统加载。
- `REC` 不要强烈闪烁，避免制造录制压力。

### 3.3 内容层

内容层负责表达阅读焦点，是最重要的运动层。

包括：

- 当前句聚光。
- 当前短语柔光扫读。
- 上一句 / 下一句的淡入淡出。
- 稿件编辑区当前段落高亮。
- 切分面板里的当前短语。

原则：

- 当前内容永远最清楚。
- 运动只帮助用户读顺，不改变阅读节奏。
- 当前短语使用「柔光扫过 + 文字微亮」，不要做传统 KTV 色块填字。
- 短语扫读应比真实朗读稍慢一点，给用户一种被陪伴而不是被追赶的感觉。

### 3.4 操作层

操作层负责按钮、卡片、hover、点击和控制栏反馈。

包括：

- 首页 `ON STAGE` 按钮。
- 卡片 hover。
- 最近稿件 hover。
- 工具按钮。
- 控制栏。
- 分段控件。
- 切分点。

原则：

- hover 只做轻微变亮、边框浮现、阴影加深或微小上浮。
- 点击反馈要短、稳，不要弹跳。
- 控制栏默认弱化，鼠标靠近时清楚浮现。
- 所有按钮动效要让用户更敢点，而不是吸引用户一直看按钮。

## 4. 关键元素原则

### 4.1 首页顶部灯光

首页顶部灯光是 Sotto 的「舞台正在被轻轻点亮」。

它可以参考 React Bits 的 `LightRays` 思路：用从画面顶部发出的低亮度光束，为首页、准备中状态和上场瞬间提供舞台感。

设计原则：

- 灯光从顶部向下散开，有明确方向。
- 亮度做柔和呼吸，周期建议 8-12 秒。
- 呼吸曲线要平滑，避免突然变亮或突然暗下去。
- 亮度变化不要影响标题和材料入口的可读性。
- 可以叠加极少量光尘漂浮，速度要慢。
- 默认来源使用 `top-center`，让它像后台上方的灯被轻轻打开。
- 色彩优先暖白、米白、低饱和淡金；只有信息状态需要时才使用很淡的青绿。
- 鼠标跟随如果开启，只允许非常弱的方向偏移，表达空气被扰动，不表达「跟随鼠标」。

推荐收敛：

- `raysOrigin` 默认使用 `top-center`。
- `raysColor` 默认使用暖白，例如 `#f3ead8` 或接近暖白的低饱和颜色。
- `raysSpeed` 建议控制在 `0.12-0.35`，不要使用默认快节奏。
- `lightSpread` 建议控制在 `0.45-0.75`，保持柔和但不过度铺满。
- `rayLength` 建议控制在 `0.8-1.2`，首页可以略长，提词窗口应更短。
- `pulsating` 可以开启，但脉冲幅度必须很低，接近呼吸。
- `fadeDistance` 建议控制在 `0.55-0.8`，光束要自然消失。
- `saturation` 建议控制在 `0.15-0.35`，避免彩色舞台感。
- `followMouse` 默认关闭；若用于首页氛围，可开启但 `mouseInfluence` 控制在 `0.02-0.06`。
- `noiseAmount` 建议控制在 `0.02-0.06`，只保留空气颗粒。
- `distortion` 建议控制在 `0.01-0.03`，避免水波感。

不应该：

- 做成舞台射灯来回扫射。
- 做成大面积白光闪烁。
- 用高饱和彩色光束。
- 让光束明显追随鼠标。
- 让 WebGL 光效盖过点阵、标题或材料入口。
- 在用户编辑或提词时持续抢眼。

### 4.2 背景点阵

点阵是 Sotto 的底层识别，也承担「空间在呼吸」的感觉。

除了静态星光点阵，也可以参考 React Bits 的 `PixelBlast` 做「像素颗粒幕布」：用低亮度、低密度、缓慢变化的像素颗粒，形成比星光点阵更柔软的背景生命感。

也可以把 Unicorn Studio 场景作为外部 WebGL 背景候选，例如项目 `xbdYsuMp0mSR01MXofEj`。它适合承载更完整的沉浸式背景，但必须被 Sotto 的低干扰原则重新约束。

默认实现优先参考 Magic UI 的 `DotPattern`：它用 SVG 提供稳定、轻量、可降级的点阵背景，是 PixelBlast 和 Unicorn Studio 之前的基础层。

设计原则：

- 点阵主要做低透明度亮度呼吸。
- 少量点可以随机微亮，像远处控制台或空气中的光尘。
- 页面切换时，点阵可以轻微视差移动，形成空间感。
- 正文区域背后的点阵要更弱，避免影响中文识别。
- DotPattern 是默认底座，适用于首页、编辑页、切分面板和提词窗口。
- DotPattern 不应完全静止，默认带非常缓慢、低幅度的呼吸感。
- DotPattern 的 `glow` 可全局存在，但要按场景分级：首页 / 准备中明显一点，编辑区 / 切分面板 / 提词窗口更弱、更慢。
- 点阵必须 `pointer-events: none`，不影响文本编辑、拖拽和点击。
- PixelBlast 背景应像像素空气，不像粒子爆炸。
- PixelBlast 可以作为首页、准备中、空状态的背景层；提词窗口中应关闭或降到几乎不可见。
- 鼠标 ripple / liquid 互动默认关闭；如用于首页，也只允许很弱的空气扰动。
- 像素形状优先 `circle` 或小 `square`，避免 triangle / diamond 带来过强图案感。
- Unicorn Studio 场景只作为首页、准备中、欢迎页或空状态背景，不用于正式提词窗口。
- Unicorn Studio 场景必须有静态降级图或纯 CSS 降级背景。
- 外部 WebGL 场景不能成为页面唯一信息来源。
- 网络失败、SDK 加载失败或用户减少动态效果时，应自动退回静态点阵 / 像素颗粒背景。

推荐节奏：

- 背景点阵整体呼吸：8-14 秒。
- 局部星点微闪：随机 2-6 秒，透明度变化很小。
- DotPattern glow 点的动画周期建议控制在 8-16 秒，随机延迟，避免整齐闪烁。
- 编辑区 / 切分面板 / 提词窗口中的 DotPattern 呼吸周期建议控制在 14-24 秒。
- 页面转场视差：200-360 ms，距离很短。
- PixelBlast `speed` 建议控制在 `0.08-0.25`。
- PixelBlast `patternDensity` 建议控制在 `0.28-0.55`。
- PixelBlast `pixelSize` 建议控制在 `2-4 px`。
- PixelBlast `edgeFade` 建议控制在 `0.35-0.6`，让边缘自然隐去。

推荐收敛：

- DotPattern `width` / `height` 建议控制在 `14-24 px`，首页可更疏，提词窗口更弱。
- DotPattern `cr` 建议控制在 `0.6-1.2`。
- DotPattern 基础透明度建议控制在 `0.12-0.28`。
- DotPattern glow 首页 / 准备中透明度峰值不超过 `0.45`，缩放不超过 `1.25`。
- DotPattern glow 编辑区 / 切分面板 / 提词窗口透明度峰值不超过 `0.28`，缩放不超过 `1.12`。
- `variant` 优先使用 `circle`；如果需要更像控制台，可使用小 `square`。
- `color` 默认使用暖白灰，例如 `#c8c1b6`、`#c0bcc4` 或低饱和淡青。
- `patternScale` 建议控制在 `2.5-5`，避免纹理过碎。
- `pixelSizeJitter` 建议控制在 `0.1-0.35`，只提供自然不均匀感。
- `enableRipples` 默认关闭；需要仪式反馈时才开启，且只响应明确点击。
- `rippleSpeed` 建议控制在 `0.12-0.25`。
- `rippleThickness` 建议控制在 `0.06-0.1`。
- `rippleIntensityScale` 建议控制在 `0.25-0.6`。
- `liquid` 默认关闭；如果开启，`liquidStrength` 控制在 `0.02-0.05`。
- `liquidWobbleSpeed` 建议控制在 `1.2-2.5`，避免水波感。
- `noiseAmount` 建议保持 `0-0.03`。
- `transparent` 默认开启，用作叠加层而不是整页底色。
- Unicorn Studio SDK 固定记录为 `2.1.11`，项目 ID 为 `xbdYsuMp0mSR01MXofEj`。
- React / Next.js 实现时可使用 `unicornstudio-react`，但必须封装成 Sotto 自己的背景组件。
- 容器必须显式设置宽高，例如 `width: 100%` 和稳定高度。
- `lazyLoad` 默认开启。
- `scale` 建议控制在 `0.5-0.85`，低性能设备降到 `0.25-0.5`。
- `dpi` 建议控制在 `1-1.5`。
- `fps` 建议控制在 `24-30`；只有独立预览或设计评审时再考虑 `60`。
- 生产环境应启用缓存相关配置，例如 `production: true`。

不应该：

- 让点阵像雨一样快速移动。
- 让点阵变成高亮噪声。
- 让 DotPattern glow 整齐同步闪烁。
- 在正文编辑区或提词窗口中开启明显、快速或高对比 glow。
- 在提词窗口正文背后形成明显干扰。
- 使用默认高强度 ripple / liquid 互动。
- 做出像素爆炸、强水波或高饱和粒子背景。
- 让 PixelBlast 和 LightRays 同时都很强；两者必须一主一弱。
- 在提词窗口里运行外部 WebGL 场景。
- 让 Unicorn Studio 场景无降级方案地依赖网络和 CDN。
- 同时叠加 LightRays、PixelBlast、Unicorn Studio 三个强动效背景。

### 4.3 状态点

状态点代表系统存在感，不代表警报。

设计原则：

- `READY.`：绿色点轻微呼吸，表示可以上场。
- `PREPARING`：白色点或暖白点慢速呼吸，表示正在准备。
- `在线`：青绿点稳定微亮，表示后台连接正常。
- `REC`：可以有极弱脉冲，但不要强红色闪烁。

推荐节奏：

- 待命 / 在线：2.4-3.2 秒一轮。
- 处理中：1.8-2.6 秒一轮。
- 录制中：3-4 秒一轮，尽量稳。

### 4.4 按钮

按钮动效应该增加「确认感」，不制造兴奋感。

首页 `ON STAGE`：

- 默认有很淡的边缘呼吸。
- hover 时边框和文字微亮，胶囊内点阵稍微清楚。
- 点击时短暂向内收 1-2 px，随后平稳进入准备状态。
- 不做弹跳、不做强光爆发。

普通按钮：

- hover：亮度提升、边框浮现、背景透明度略升。
- active：轻微压下。
- disabled：保持安静灰，不要闪烁提醒。

提词控制按钮：

- 默认低透明度。
- 鼠标进入提词卡片时整体浮现。
- hover 单个按钮时只强调该按钮，不改变正文亮度。

### 4.5 卡片

卡片是舞台里的工作台，动效要有重量。

卡片材质可以参考 React Bits 的 `ReflectiveCard`：用暗色玻璃、轻微反射、边缘高光和细噪声，让主要容器像后台控制台里的黑色玻璃面板。但 Sotto 不采用原组件里的 webcam 背景、身份卡信息和强安防风格。

可滚动卡片和长内容区域可以参考 Magic UI 的 `ProgressiveBlur`：用顶部 / 底部渐进模糊让内容边界更柔和，同时暗示还可以继续滚动。

设计原则：

- 页面进入时卡片淡入 + 极短距离上浮。
- hover 时卡片边框微亮，背景透明度略升。
- 当前选中卡片可有柔光边框，但不能频繁闪。
- 卡片之间不要互相抢焦点。
- 反光高光只沿边缘或斜向轻轻掠过，不覆盖正文。
- 材质噪声要非常细，像磨砂玻璃，不像脏污纹理。
- 背景折射 / 位移只用于卡片边缘，不让文字区域变形。
- 不默认请求摄像头权限，不把用户视频作为卡片材质。
- 滚动边缘渐进模糊只放在可滚动内容的顶部 / 底部，不遮挡当前焦点内容。
- 渐进模糊层必须 `pointer-events: none`，不影响文本选择、点击和拖拽。
- 内容滚动到顶部或底部时，对应方向的模糊可以减弱或消失。

推荐节奏：

- 卡片进入：240-420 ms。
- hover 反馈：120-180 ms。
- 选中态稳定：持续柔光，不做循环强闪。

推荐收敛：

- `blurStrength` 建议控制在 `2-6 px`，不要使用过重模糊。
- `metalness` 建议控制在 `0.12-0.35`，只保留轻微反光。
- `roughness` 建议控制在 `0.18-0.35`，形成细腻磨砂感。
- `displacementStrength` 建议控制在 `2-6`，只影响边缘或背景层。
- `glassDistortion` 建议控制在 `1-4`，避免玻璃扭曲。
- `noiseScale` 建议控制在 `1-2`。
- `specularConstant` 建议控制在 `0.5-1.0`，避免亮斑过硬。
- `grayscale` 可保持 `1`，维持暗色低饱和。
- `overlayColor` 建议使用 `rgba(0, 0, 0, 0.28-0.48)` 或低亮暖白薄层。
- `ProgressiveBlur` 默认高度建议控制在容器高度的 `12%-24%`，长列表可到 `30%`。
- `position` 根据滚动方向使用 `top`、`bottom` 或 `both`，不要无脑全覆盖。
- `blurLevels` 建议使用更克制的层级，例如 `[0.5, 1, 2, 4, 8, 12, 16, 24]`，避免默认最高 `64 px` 造成雾化。
- 提词窗口中最高 blur 不建议超过 `8-12 px`。

不应该：

- 使用 webcam / camera feed 作为默认视觉来源。
- 做成金属证件、门禁卡或安防仪表盘。
- 让卡片反光比当前句聚光更亮。
- 让卡片内容因为滤镜位移而不可读。
- 在一页中放太多反光卡片，造成视觉噪声。
- 用渐进模糊遮挡当前句、当前短语或主要操作。
- 让整块内容都处在 blur 下，降低阅读速度。

### 4.6 场合型公告卡片

场合型公告卡片可以参考 Linear 宣布新 UI 的动态卡片思路：当产品有一个值得用户注意、但不需要打断工作的消息时，用一张精致、短暂、有状态感的卡片轻轻宣布。

Sotto 采用它的「漂亮地出现一次」的场合感，不采用营销弹窗或强运营提醒。

适用位置：

- 第一次进入 Sotto 时的欢迎 / 气质说明。
- 新版本或重要 UI 更新提示。
- 稿件自动切分完成后的轻量完成卡片。
- 从准备中进入编辑页时的「节奏已整理好」提示。
- 成功导出稿件或保存预设时的低频反馈。

设计原则：

- 公告卡片必须有明确场合，不用于普通状态更新。
- 出现时应从当前页面语境中浮现，而不是从屏幕外突然飞入。
- 卡片内容要短，优先一句主信息 + 一个轻操作。
- 动效可以更精致一点，但仍然安静、短促、低干扰。
- 卡片应自动退场或容易关闭，不能阻塞用户继续工作。
- 卡片视觉可以带反光玻璃、点阵、轻微光束，但不要同时堆满。

推荐收敛：

- 进入：320-520 ms。
- 停留：根据场景控制在 2.5-5 秒，除非需要用户确认。
- 退场：220-360 ms。
- 背景弱化：透明度变化控制在 `0.08-0.18`。
- 卡片位移控制在 `8-20 px`。
- 高光扫过最多一次，不循环。

不应该：

- 做成营销弹窗、公告横幅或强制 modal。
- 每次完成小操作都弹出公告卡片。
- 用大面积彩色渐变宣布状态。
- 让公告卡片遮挡当前句或提词窗口。
- 在录制 / 提词中自动出现公告。

### 4.7 切分点与节奏控件

切分点要像节奏标记，不像表单控件。

分段控件和模式选项可以参考 AllTrails 选择筛选项时的 icon micro-animations：当用户选择一个筛选 / 模式时，图标用一次短促的语义动作确认状态，而不是只改变颜色。

设计原则：

- hover 切分点时，节点和竖线轻微亮起。
- 点击插入切分点时，用短促柔光确认。
- 取消切分点时，光线收回，不要突然消失。
- 停顿 / 强调控件的选中态可以带一条缓慢流动的细光，但不能抢当前句。
- 图标微动效只在选中状态变化时触发一次。
- 图标动作应贴合含义：停顿像轻轻停住，强调像微亮加重，节奏像短波展开。
- 未选中的图标保持安静，不持续循环吸引注意。

适用位置：

- 停顿设置：无 / 短停顿 / 长停顿。
- 强调设置：普通 / 强调。
- 速度模式：舒缓 / 标准 / 紧凑。
- 右侧步骤节点：短语切分 / 状态调整 / 节奏控制。

推荐收敛：

- 图标选中动效：180-320 ms。
- 缩放控制在 `1.0-1.08`。
- 位移控制在 `1-3 px`。
- 旋转控制在 `0-8deg`，只在图标语义需要时使用。
- 亮度变化比尺寸变化更重要。

不应该：

- 让所有筛选图标同时动。
- 让图标循环播放。
- 用夸张弹跳表达选中。
- 让图标动效比文字选中态更显眼。
- 在提词窗口控制栏里使用复杂图标动画。

### 4.8 点击确认火花

点击确认火花可以参考 React Bits 的 `ClickSpark` 思路：在少数关键点击后，出现短促、低亮、向外散开的线性微光，表达「动作被 Sotto 收到了」。

Sotto 采用它的确认感，不采用庆祝感。

适用位置：

- 首页 `ON STAGE` 点击。
- 切分点插入成功。
- 准备中某一步完成并进入下一步。
- 提词窗口开始 / 继续。
- 保存、导出等低频明确动作。

设计原则：

- 火花只出现在有意义的确认动作上，不做全局点击特效。
- 火花颜色优先暖白、淡金或低饱和青绿。
- 火花应像光尘短暂散开，而不是礼花、爆炸或游戏反馈。
- 对取消、删除、错误操作不使用火花，避免误导情绪。
- 提词窗口中只允许极弱火花，不能打断阅读。

推荐收敛：

- `sparkColor` 使用 `#f3ead8`、`#d8d0bd` 或低饱和青绿。
- `sparkSize` 建议控制在 `4-8 px`，不要使用默认大尺寸。
- `sparkRadius` 建议控制在 `12-22 px`。
- `sparkCount` 建议控制在 `4-6`。
- `duration` 建议控制在 `180-280 ms`。
- `easing` 使用 `ease-out`。
- `extraScale` 建议控制在 `0.6-0.9`。

不应该：

- 包裹整页，让每次点击都冒火花。
- 在 hover 时触发火花。
- 在连续拖动、滑杆、正文编辑时触发火花。
- 使用红色、彩虹色或高饱和火花。
- 让火花持续超过一次眨眼。

### 4.9 弹性节奏滑杆

弹性滑杆适合用于 Sotto 的速度、停顿强度、节奏密度等「表达节奏」调节。它可以参考 React Bits 的 `ElasticSlider` 思路：拖动时轨道有轻微弹性，拖到边界外时有阻尼，松手后平稳回正。

Sotto 采用它的交互思想，而不是原样采用夸张的视觉强度。

适用位置：

- 稿件编辑页的速度滑杆。
- 提词预览区的节奏微调。
- 提词窗口里的 `- / 0.9x / +` 速度反馈。
- 切分面板里的停顿强度或强调强度。

设计原则：

- 滑杆表达的是「语速和呼吸」，不是音量或数值控制。
- 左右图标优先使用节奏语义，例如「更慢 / 更快」「更松 / 更紧」，不要直接套用音量图标。
- 拖动时轨道可以变厚、微亮，给用户明确的可控感。
- 超出边界时可以有轻微拉伸和阻尼，暗示已经到达舒缓 / 紧凑边界。
- 松手后轨道回正要柔和，不要有明显弹跳。
- 数值显示应转译为用户语言，例如 `舒缓`、`标准`、`紧凑` 或 `0.9x`，避免暴露复杂参数。

推荐收敛：

- 原组件 hover 放大到 `1.2` 对 Sotto 偏强，建议控制在 `1.03-1.08`。
- 原组件边界图标放大到 `1.4` 偏玩具感，建议控制在 `1.08-1.15`。
- 原组件 `bounce: 0.5` 偏弹，建议降低到 `0.12-0.25`。
- `MAX_OVERFLOW` 应根据控件宽度收敛，视觉溢出不超过轨道长度的 8%-12%。
- 轨道高度变化应轻微，例如默认 4-6 px，交互中 6-9 px。
- value indicator 默认弱化，拖动时浮现，松手后淡出或回到低亮。

不应该：

- 让滑杆像橡皮筋一样大幅形变。
- 让图标在边界处弹跳吸睛。
- 把速度调节做成强数值仪表盘。
- 在正式提词窗口里持续显示复杂滑杆，影响当前句阅读。

### 4.10 窗口边界橡皮筋

窗口边界橡皮筋可以参考 Things 在窗口缩到最小尺寸以下时的 rubberbanding 效果：当用户继续拖拽一个已经到达边界的窗口或面板时，界面不应生硬卡死，而是给出轻微、有阻尼的回弹反馈。

Sotto 采用它的「边界有弹性」手感，不采用夸张的弹跳。

适用位置：

- 正式提词窗口调整宽度、高度或字号到边界时。
- 提词窗口拖到屏幕边缘时。
- 切分面板宽度到最小 / 最大值时。
- 底部提词预览卡片被拉伸到边界时。
- 首页材料入口高度达到最小可用尺寸时。

设计原则：

- 边界反馈应告诉用户「这里已经到头了」，而不是让用户以为操作失败。
- 橡皮筋只发生在拖拽 / resize 的边界瞬间，不用于普通 hover 或点击。
- 到达边界后可以继续产生 4-10 px 的视觉阻尼位移，松手后回到合法尺寸。
- 回弹应短而柔，保持 Mac 原生感。
- 正文区域不能被明显压缩变形；变形优先作用在外壳、边缘和阴影上。
- 边界到达时可以伴随边框微亮或极轻触觉反馈，避免只靠形变表达。

推荐收敛：

- 边界溢出视觉位移：`4-10 px`。
- 回弹时长：`160-260 ms`。
- resize 边缘高光：`120-180 ms`。
- 回弹曲线使用低弹性的 spring 或 `ease-out`，避免过冲。
- 最小尺寸必须优先保护可读性：提词窗口不能缩到下 3 句不可读。

不应该：

- 让整个窗口像橡皮布一样大幅拉伸。
- 在正文文字上做非等比压缩。
- 到达边界时完全无反馈。
- 用警告色表达普通尺寸边界。
- 让回弹持续太久，影响录制前调整窗口的效率。

### 4.11 产品名轻微信号漂移

产品名和状态字标可以参考 React Bits 的 `FuzzyText` 思路：用很轻的水平像素漂移，让 `Sotto` 像暗场里被微弱信号点亮的名字。

Sotto 采用它的「微弱生命感」，不采用强故障美学。

适用位置：

- 首页大号 `Sotto` 产品名。
- 处理中状态里的 `Sotto` 小标识。
- 稿件编辑页右上角 `SOTTO`。
- 提词窗口左侧 `SOTTO PROMPT`。
- 极少量状态字，例如 `READY.`、`PREPARING`，但必须保证识别。

设计原则：

- 模糊感应像点阵空气在轻轻漂移，而不是屏幕坏掉。
- 默认状态下产品名必须清楚可读。
- hover 时可以略微增加漂移，或反过来变得更清晰，表达「被触碰后被听见」。
- 方向优先使用水平漂移，避免垂直抖动影响中文或品牌字识别。
- 只用于短字标，不用于长句、正文稿件、提词内容和关键操作按钮。

推荐收敛：

- 原组件 `baseIntensity: 0.18` 对 Sotto 偏强，建议控制在 `0.03-0.08`。
- 原组件 `hoverIntensity: 0.5` 明显偏 glitch，建议控制在 `0.08-0.14`。
- `fuzzRange` 建议控制在 `4-10 px`，不要使用默认 `30 px`。
- `fps` 可降到 `24-30`，既省资源，也更像低频呼吸。
- `direction` 优先使用 `horizontal`。
- `transitionDuration` 建议大于 `0`，让 hover 变化有过渡。
- `clickEffect` 默认关闭。
- `glitchMode` 默认关闭，除非未来有明确的错误或断连状态。
- `gradient` 默认不用，保持暖白 / 低亮点阵品牌字。

不应该：

- 把 Sotto 产品名做成持续故障抖动。
- 在首页主标题之外大量使用模糊字。
- 让中文提词正文产生 fuzzy 效果。
- 使用点击爆发制造游戏化反馈。
- 用彩色渐变削弱 Sotto 的暗色微光气质。

### 4.12 当前短语扫读

这是 Sotto 提词体验的核心动效。

设计原则：

- 扫读不是填色，而是光从短语背后柔和经过。
- 当前短语文字略微变亮、边缘更清晰。
- 光扫过后，短语回到当前句的正常亮度，不突然变暗。
- 高亮边界要柔，不要像矩形色块。
- 对中文而言，扫读应以短语为单位，而不是逐字追踪。

推荐表现：

- 光带宽度略大于短语。
- 光带透明度低，边缘羽化。
- 文字亮度变化比背景光更明显。
- 扫读速度跟随速度滑杆，但设下限，避免太快。

## 5. 页面流转

Sotto 的页面流转应像「从后台走向上场」。

可以参考 Luma 打开 / 关闭活动详情的平滑转场思路：用户点击的卡片不是消失后切到新页面，而是成为下一层界面的视觉来源；关闭时也回到原来的位置。Sotto 里这类转场应强调「来源连续性」，让用户始终知道自己从哪段稿件、哪个卡片、哪个预览进入了当前状态。

### 5.0 来源连续转场

来源连续转场适用于「从一个已有对象进入更专注的详情状态」。

适用位置：

- 最近稿件卡片 -> 稿件编辑页。
- 当前句 -> 切分面板。
- 提词预览卡片 -> 正式提词窗口。
- 首页材料入口 -> 准备中状态。

设计原则：

- 被点击对象应成为下一层界面的视觉起点。
- 打开时，来源卡片可以轻微放大、移位、扩展为新容器。
- 关闭时，容器应柔和缩回或淡回原位置，而不是突然消失。
- 背景层可以轻微变暗或退后，帮助用户理解层级变化。
- 标题、当前句、状态点等可作为 shared element 保持连续。
- 转场期间正文可短暂弱化，但不能闪烁或重排。

推荐收敛：

- 打开：320-520 ms。
- 关闭：240-420 ms，略快于打开。
- 背景遮罩淡入 / 淡出：160-260 ms。
- 来源对象缩放控制在 `1.0-1.04`，避免弹窗感。
- 位移路径应短而明确，优先从来源位置移动到目标位置。

不应该：

- 先清空页面再显示新页面。
- 打开和关闭使用完全无关的动画。
- 让卡片像弹窗一样从中心凭空出现。
- 在关闭时丢失用户刚才所在的稿件 / 句子位置。

### 5.1 首页 -> 准备中

含义：用户把稿件交给 Sotto，系统开始整理节奏。

动效建议：

- `ON STAGE` 点击后短暂压下。
- 首页主卡片的光线向中心收拢。
- 页面淡入准备中卡片。
- 顶部状态从 `READY.` 平滑切到 `PREPARING`。
- 三个准备步骤依次浮现。

避免：

- 全屏 loading 遮罩。
- 转圈加载器。
- 大幅页面翻转。

### 5.2 准备中 -> 稿件编辑

含义：稿件已经从材料变成可调整的提词节奏。

动效建议：

- 底部声波或点阵节奏线轻轻收束。
- 当前完成步骤微亮后退场。
- 稿件编辑区从中间展开，右侧面板随后浮现。
- 自动保存状态稳定出现。
- 如果准备中容器来自首页材料入口，编辑页主稿件区域应继承这个容器的位置和宽度关系。
- 可以出现一次短暂的场合型公告卡片，例如「提词节奏已整理好」，但必须快速退场，不阻塞编辑。

避免：

- 突然切到复杂编辑界面。
- 让用户感觉进入了字幕软件。

### 5.3 稿件编辑 -> 切分面板

含义：从全稿准备进入单句节奏微调。

右侧辅助面板可以参考 Typefully 的 smooth sliding sidebars：侧边栏应像工作台的一部分顺滑滑入和切换，而不是突然弹出一个独立抽屉。

动效建议：

- 被选中的句子柔光边框稳定亮起。
- 右侧面板或详情页面从同一位置扩展。
- 当前句在面板中放大呈现。
- 切分点沿句子节奏逐个浮现。
- 关闭切分面板时，当前句应回到稿件中的原位置，并保留短暂柔光帮助定位。
- 侧边栏打开时主稿件保持稳定，只做轻微让位或焦点弱化。
- 在「短语切分 / 状态调整 / 节奏控制」之间切换时，同一侧栏容器保持不变，内容横向或纵向轻滑切换。
- 侧栏关闭时应沿原方向退回，不要直接透明消失。

推荐收敛：

- 侧栏打开：260-420 ms。
- 侧栏关闭：220-360 ms。
- 侧栏内切换：180-280 ms。
- 侧栏位移距离以自身宽度为基准，但末端要缓慢落位。
- 主稿件让位位移控制在 `0-16 px`，避免用户丢失阅读位置。

避免：

- 整页大幅跳转。
- 选中句失去空间来源，让用户不知道自己在调哪句。
- 侧栏覆盖主稿件关键文字。
- 多个侧栏层层叠加。

### 5.4 编辑 / 切分 -> 提词窗口

含义：从准备台进入正式上场。

动效建议：

- 提词预览区中的卡片成为正式提词窗口的视觉来源。
- 当前句保持连续，不要重新从空白出现。
- 控制栏先弱化出现，当前句随后聚光。
- 左侧 `SOTTO PROMPT` 状态栏稳定亮起。
- 退出提词窗口时，提词卡片应回到编辑页底部预览区，而不是直接关闭成空白。

避免：

- 弹窗突然砸出来。
- 先展示控制按钮，再展示正文。
- 用强烈倒计时制造紧张感。

### 5.5 提词暂停 / 继续

含义：表达暂时停住，但空间还在。

动效建议：

- 暂停后当前短语扫读停住或慢慢收光。
- 背景呼吸继续，但速度更慢。
- 控制栏更清楚，提示用户可以继续。
- 继续时不要跳读，光从当前短语平稳恢复。

避免：

- 暂停时全界面冻结成死状态。
- 继续时高亮突然跳到下一句。

### 5.6 明暗模式切换（未来）

明亮 / 黑暗模式切换可以参考 Magic UI `AnimatedThemeToggler`：用 View Transitions API 的 clip-path reveal，在主题变化时让新主题从一个明确来源展开。

本期不做明暗模式切换，只把它作为后续设计储备。

适用原则：

- 主题切换不是普通页面转场，而是一次全局「换光」。
- 默认形态优先采用 `circle`，也可评估更安静的 `rectangle`；避免 `star`、`triangle` 等强装饰形状。
- 展开来源优先来自主题按钮，让用户知道触发点；如果未来主题切换放在欢迎页或设置页，也可以从视口中心展开。
- 转场时长建议控制在 360-520 ms，足够柔和，但不能拖慢进入提词。
- 切换过程中不要改变正文布局、当前句位置和提词窗口尺寸。
- 如果浏览器 / WebView 不支持 `document.startViewTransition`，应直接切换主题，不显示错误。

不应该：

- 把主题切换做成炫技全屏特效。
- 使用多边形快速扫过，制造强烈视觉噪声。
- 在正式提词过程中自动触发主题切换。
- 把主题 reveal 复用到稿件编辑、侧栏打开或提词状态变化。

## 6. 微交互清单

### 首页

- 顶部舞台光极慢呼吸，可使用低强度 `LightRays`。
- 点阵背景默认可用低透明度、慢呼吸的 `DotPattern`，并可替换为低密度 `PixelBlast` 像素颗粒幕布。
- 如使用 Unicorn Studio 场景，只作为低干扰背景层，并保留静态降级。
- `Sotto` 产品名可以有极轻的水平信号漂移，hover 时微微增强或变清晰。
- `READY.` 状态点稳定微光。
- 材料入口 hover 时边框微亮。
- `ON STAGE` 按钮 hover 时文字、边框和侧边点阵微亮。
- `ON STAGE` 点击时可出现一次极轻确认火花，随后进入准备中。
- 最近稿件 hover 时行背景轻微浮现，状态点保持稳定；打开稿件时卡片应连续扩展到编辑页。

### 准备中

- `PREPARING` 状态点慢速呼吸。
- 顶部光束比首页更收敛，像后台灯光正在等待，不像正式开场。
- 背景可以使用低密度 PixelBlast，让处理中状态像稿件正在被安静整理成节奏。
- Unicorn Studio 场景如用于准备中，必须比首页更暗、更慢，避免干扰处理中步骤。
- 标题下方 3 个点依次轻亮，节奏慢于普通 loading。
- 当前步骤节点发光，后续节点弱化等待。
- 完成当前步骤时可以有极小确认火花或节点微光扩散。
- 底部点阵声波缓慢起伏。
- 完成某步时光从当前节点传到下一节点。
- 全部完成后可用一张短暂公告卡片轻轻提示「节奏已整理好」。

### 稿件编辑

- 当前段落边框柔光稳定亮起。
- 当前段落或底部预览卡片可带轻微反光玻璃边缘，但正文区域保持清楚。
- 右上角 `SOTTO` 标识保持低亮，可有极轻微信号漂移。
- 右侧步骤节点 hover 时只亮当前项；切换步骤时侧栏内容顺滑滑动，不硬切。
- 稿件编辑区滚动时，顶部 / 底部可用渐进模糊提示内容延续。
- 底部提词预览中的当前句保持轻微扫读。
- 速度滑杆拖动时轨道轻微变厚，边界阻尼提示「已经很舒缓 / 很紧凑」。
- 自动保存状态用文字变化和小点微亮表达，不用 toast 打扰。
- 工具栏按钮 hover 低调浮现。

### 切分面板

- 当前句进入时淡入，不逐字飞入。
- 长内容区域可用顶部 / 底部渐进模糊保护滚动边界。
- 切分点 hover 时节点和竖线亮起。
- 插入 / 取消切分点时有短促柔光确认。
- 插入切分点成功时可出现一次小范围确认火花。
- 停顿控件选中态带稳定光圈，图标可做一次「轻轻停住」的微动效。
- 强调控件选中态带细微波线，图标可做一次「微亮加重」的微动效。
- 停顿 / 强调滑杆可有轻微弹性，但只表达手感，不制造玩具感。
- 当前句预览同步展示柔光扫读。

### 提词窗口

- 当前句最大最亮，保持稳定。
- 左侧 `SOTTO PROMPT` 保持稳定低亮，可有极弱漂移，但不能抢当前句。
- 提词卡片外壳可以有轻微玻璃反光，反光只服务边界和舞台质感。
- 调整窗口尺寸到最小 / 最大边界时，外壳可有轻微橡皮筋回弹，但正文不变形。
- 上一句 / 未来句边缘可用极轻渐进模糊退后，但当前句不能被模糊覆盖。
- 上一句和未来句通过透明度建立层级，不频繁移动。
- 当前短语柔光扫读。
- 底部节奏线跟随进度轻微推进。
- 控制栏默认弱化，鼠标靠近时浮现。
- 速度调整只用局部数值、轻微弹性和节奏线反馈，不打断正文。

## 7. 实现口径建议

下面是后续实现时可以采用的默认口径，具体数值可在原型中微调。

### 时间

- hover：120-180 ms。
- 点击反馈：80-140 ms。
- 点击确认火花：180-280 ms。
- 图标选中微动效：180-320 ms。
- 产品名字标 hover 漂移过渡：180-320 ms。
- 场合型公告卡片进入：320-520 ms。
- 场合型公告卡片退场：220-360 ms。
- 来源连续转场打开：320-520 ms。
- 来源连续转场关闭：240-420 ms。
- 侧边面板打开：260-420 ms。
- 侧边面板关闭：220-360 ms。
- 侧边面板内容切换：180-280 ms。
- 滑杆拖动反馈：跟手即时反馈，轨道厚度变化控制在 80-140 ms 内。
- 滑杆松手回正：180-320 ms。
- 窗口边界橡皮筋回弹：160-260 ms。
- 卡片淡入：240-420 ms。
- 页面转场：280-520 ms。
- 未来主题切换：360-520 ms。
- 状态点呼吸：1.8-4 秒。
- 背景灯光呼吸：8-12 秒。
- 光束纹理流动：低速持续，等效周期不短于 10 秒。
- 点阵生命感：8-14 秒。
- PixelBlast 像素幕布：低速持续，等效周期不短于 12 秒。

### 曲线

优先使用平滑、自然的缓动：

- `ease-out`：用于元素进入。
- `ease-in-out`：用于呼吸、状态切换、页面转场。
- 未来主题切换 reveal 可使用 `ease-in-out`，保持像换光而不是扫屏。
- 避免弹性曲线、回弹曲线和过冲。

### 幅度

- hover 位移控制在 1-3 px。
- 来源连续转场缩放控制在 `1.0-1.04`。
- 侧边面板引起的主稿件让位控制在 `0-16 px`。
- 图标选中微动效缩放控制在 `1.0-1.08`，位移控制在 `1-3 px`。
- 窗口 resize 边界溢出控制在 `4-10 px`，且只作用于外壳和边缘。
- 顶部光束速度控制在 `0.12-0.35`，鼠标影响控制在 `0.02-0.06` 或关闭。
- 顶部光束噪声控制在 `0.02-0.06`，扭曲控制在 `0.01-0.03`。
- 顶部光束饱和度控制在 `0.15-0.35`。
- PixelBlast 速度控制在 `0.08-0.25`，密度控制在 `0.28-0.55`。
- DotPattern 间距控制在 `14-24 px`，半径控制在 `0.6-1.2`，基础透明度控制在 `0.12-0.28`，默认保持慢呼吸。
- PixelBlast ripple 强度控制在 `0.25-0.6`，默认关闭。
- PixelBlast liquid 强度控制在 `0.02-0.05`，默认关闭。
- ClickSpark 火花半径控制在 `12-22 px`，数量控制在 `4-6`，尺寸控制在 `4-8 px`。
- 场合型公告卡片位移控制在 `8-20 px`，背景弱化控制在 `0.08-0.18`。
- ReflectiveCard 材质应低反光：`metalness 0.12-0.35`、`blurStrength 2-6 px`、`displacementStrength 2-6`。
- 玻璃扭曲控制在 `glassDistortion 1-4`，并避开正文区域。
- ProgressiveBlur 高度控制在容器高度的 `12%-24%`，长列表最高约 `30%`。
- ProgressiveBlur 最高 blur 在普通滚动区控制在 `16-24 px`，提词窗口控制在 `8-12 px`。
- 产品名字标默认 fuzzy intensity 控制在 `0.03-0.08`，hover 控制在 `0.08-0.14`。
- 产品名字标 `fuzzRange` 控制在 `4-10 px`，优先水平漂移。
- 滑杆 hover / 拖动缩放控制在 `1.03-1.08`。
- 滑杆边界图标反馈控制在 `1.08-1.15`。
- 滑杆边界溢出不超过轨道长度的 8%-12%。
- 卡片进入位移控制在 8-16 px。
- 光效透明度变化要小于文字透明度变化。
- 背景层运动距离要非常短，宁可几乎不可见。

### 可访问性与降噪

- 必须尊重系统的「减少动态效果」设置。
- 降低动态效果时，保留状态变化和聚光层级，但取消持续呼吸与背景闪烁。
- 如果使用 WebGL 光束，应只在可见区域运行，并在页面不可见或窗口失焦时暂停或降帧。
- 降低动态效果时，`LightRays` 应退化为静态柔光背景。
- 如果使用 PixelBlast，应启用离屏暂停；减少动态效果时退化为静态低亮点阵或纯暗色颗粒背景。
- 如果使用 Unicorn Studio，应懒加载、可见时运行、不可见时销毁或暂停；减少动态效果时退化为静态背景。
- 外部 SDK 加载失败不能阻塞页面主要操作。
- 未来明暗模式切换必须有 `document.startViewTransition` 不可用时的直接切换降级。
- DotPattern 是默认降级背景；减少动态效果时可以关闭 glow，保留低亮点阵。
- 如果使用反光玻璃卡片，减少动态效果时保留静态边框和透明度，不播放高光扫过。
- ProgressiveBlur 是静态阅读辅助层，减少动态效果时可以保留，但应避免过高 blur。
- 提词窗口中，所有持续动效都不能影响中文识别速度。
- 录制场景下应避免过密动效，减少观众注意力被提词层吸走。

## 8. 禁止清单

Sotto 不应出现以下动效：

- KTV 式强色块逐字填充。
- 大面积霓虹渐变流动。
- 高饱和彩色粒子喷发。
- 按钮弹跳、果冻回弹。
- 页面 3D 翻转。
- 页面硬切、空白切换或来源位置丢失的详情打开方式。
- 侧栏硬切、层层叠加或覆盖主稿件关键文字。
- 强烈 glitch 抖动。
- 产品名持续高强度 fuzzy / glitch。
- 正文稿件 fuzzy 化。
- 高频闪烁状态点。
- 一直转圈的 loading。
- 高速流动或高饱和的光束。
- 明显追随鼠标的舞台光。
- 高亮、同步闪烁的 DotPattern glow。
- 高密度、高速度的 PixelBlast 粒子背景。
- 默认开启强 ripple / liquid 像素互动。
- 外部 WebGL 场景无懒加载、无降级、无性能上限。
- 提词窗口中运行 Unicorn Studio 背景。
- 全局点击火花。
- 高饱和、长时长或大半径 ClickSpark。
- 默认调用摄像头的反光卡片。
- 强金属、强安防、强扭曲的 ReflectiveCard。
- 用 ProgressiveBlur 遮挡当前焦点内容或主要操作。
- 高强度 blur 造成稿件正文雾化。
- 高频出现的公告卡片。
- 录制 / 提词中自动弹出的公告卡片。
- 循环播放或同时播放的筛选图标动画。
- 提词控制栏里的复杂图标动画。
- 让正文或卡片持续漂移。
- 橡皮筋式大幅弹性滑杆。
- 正式提词窗口里常驻复杂滑杆。
- 大幅拉伸窗口、面板或正文的 rubberbanding。
- 到达最小尺寸时无反馈或使用警告色反馈。
- 用星形、三角形等强装饰 reveal 做默认主题切换。
- 在正式提词过程中自动触发明暗主题切换。
- 提词时让背景动效比文字更显眼。

## 9. 验收标准

后续做原型或实现时，可以用这组问题检查：

- 首页不操作时，界面是否像安静呼吸，而不是静态海报？
- 顶部灯光是否柔和、有方向、不抢标题？
- 顶部光束是否像舞台被轻轻点亮，而不是炫技背景？
- 背景点阵是否有生命感，但不影响文字？
- 默认 DotPattern 是否有缓慢呼吸感，同时足够安静，并能作为其他背景失败时的可靠降级？
- 如果使用 PixelBlast，它是否像像素空气，而不是粒子特效？
- 如果使用 Unicorn Studio，它是否只是背景氛围，而不是抢走主任务？
- 状态点是否表达「系统在场」，而不是警报？
- 点击火花是否只出现在关键确认动作，而不是变成全局装饰？
- 反光玻璃卡片是否提升质感，而不是抢走正文注意力？
- 渐进模糊是否帮助理解滚动边界，而不是降低可读性？
- 公告卡片是否只在真正有场合时出现，而不是打断工作？
- 图标微动效是否只在选中时短促确认，而不是持续抢注意力？
- hover 是否让用户更敢点击，而不是吸引用户玩按钮？
- 产品名 fuzzy 是否仍然清楚、安静，而不是故障感？
- 速度 / 节奏滑杆是否有柔和手感，而不是玩具式弹跳？
- 准备中是否像「上场前整理节奏」，而不是普通 loading？
- 页面流转是否保留空间连续性？
- 打开和关闭详情时，用户是否始终知道它来自哪个卡片 / 句子 / 预览？
- 侧边栏打开、关闭和切换时，主稿件是否仍然稳定可定位？
- resize 到边界时，用户是否感到边界明确但不生硬？
- 未来明暗模式切换是否像一次安静换光，而不是炫技扫屏？
- 提词窗口里当前句是否永远最清楚？
- 当前短语扫读是否像被光照亮，而不是被色块覆盖？
- 用户暂停时，界面是否仍然安心、可继续？
- 所有动效是否都能一句话解释其用途？

## 10. 一句话原则

> Sotto 的动效不是表演给用户看，而是在用户表达之前，替他把舞台轻轻点亮。
