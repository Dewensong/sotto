# Sotto UI / 交互差距评审与 V2 改造方案

> 日期：2026-05-03  
> 背景：MVP 0.1 已完成 SwiftPM + SwiftUI + AppKit 的可运行闭环，但 Dewens 反馈当前 UI / 交互与预期相差甚远。本文件用于重新对齐 Sotto 的界面气质、交互结构和下一轮改造重点。

## 1. 总体判断

当前实现完成了技术闭环，但没有真正完成 Sotto 的产品体验闭环。

它现在更像：

- 一个暗色主题的本地提词工具。
- 一个可以粘贴、切分、预览、打开浮窗的工程原型。
- 一个验证数据流和窗口能力的底座。

但 Sotto 的目标应该是：

- 一个安静、精致、有仪式感的演出后台。
- 一个把稿件整理成表达节奏的准备空间。
- 一个让用户在上场前更安心、更稳、更有掌控感的 AI 演示导演。

核心差距不是颜色、字体或圆角几个参数，而是 **页面叙事、视觉层级、交互节奏和状态感都没有跟上参考图**。

## 2. 当前实现的主要问题

### 2.1 首页：像输入工具，不像演出后台

当前问题：

- 页面结构是「左侧大输入框 + 右侧最近稿件」，很直接，但缺少仪式感。
- `Sotto` 标识是普通大字，没有点阵 / 舞台屏 / 控制台符号感。
- 主入口是普通 `TextEditor`，不像“把稿子放进后台”的材料入口卡片。
- 缺少顶部舞台光、状态区、文档图标、能力提示、节奏线和底部标语。
- `ON STAGE` 按钮尺寸、质感和中心性不足，不像上场按钮。

目标状态：

- 首页应优先还原 `sotto-homepage-v1.png` 的空间结构：顶部品牌与状态、中央标题、中心材料入口、大号仪式 CTA、下方最近稿件。
- 材料入口应像一个舞台后台的透明容器，而不是普通文本输入框。
- `ON STAGE / 准备上场` 应是首页唯一强主按钮，带发光边框、点阵端头和节奏线前奏。

V2 改法：

- 首页改为纵向舞台布局，而不是左右分栏工具布局。
- 顶部保留左侧品牌、右侧 `READY.` 状态点和设置入口。
- 中央使用点阵风格标题和说明。
- 主输入区改为大号材料入口卡片：空状态显示文档图标、能力提示和粘贴引导；聚焦后再进入编辑态。
- 最近稿件移到主入口下方，用三行列表卡片展示状态点、更新时间和更多菜单。

### 2.2 准备中状态：当前缺失

当前问题：

- 用户点击 `ON STAGE` 后直接跳编辑页。
- 缺少“正在整理这段稿子的提词节奏”的过渡状态。
- 系统没有表现“后台正在工作”，导致产品人格弱。

目标状态：

- 点击上场后进入短暂准备中状态，即使确定性切分很快，也应有 600-900ms 的舞台化过渡。
- 状态应参考 `sotto-preparing-state-v1.png`：顶部 `PREPARING`、中央处理文案、三步节点、底部光波。

V2 改法：

- 在 AppModel 中引入 `appPhase`: `home / preparing / editor`。
- `preparing` 先展示 3 个步骤：识别段落、切分句子、生成短语高亮节奏。
- 对确定性切分使用最短展示时长，不阻塞真实处理，但保留仪式感。

### 2.3 编辑页：像普通编辑器，不像稿件准备台

当前问题：

- 主稿件区是一个大 `TextEditor`，缺少段落编号、当前段落高亮、自动保存状态。
- 句子 chip 横向滚动像调试辅助，不像稿件结构。
- 顶部信息拥挤，Sotto 品牌感缺失。
- 右侧切分面板太像字符调试器。
- 底部预览虽然存在，但没有正式提词窗口的左侧状态栏、波形线和控制台比例。

目标状态：

- 编辑页应优先还原 `sotto-script-editor-v1.png`：左侧稿件准备台、右侧垂直功能面板、底部嵌入式提词预览。
- 主稿件应按句/段落列表展示，每个块保留可编辑能力，当前选中块有发光边框和清楚层级。
- 顶部显示返回、稿件标题、自动保存状态、右上角 `SOTTO` 与打开提词窗口。

V2 改法：

- 从单个大 `TextEditor` 改为「段落 / 句子块列表」。
- 每个句子块展示编号、正文、切分边界；选中块有发光边框和右侧状态点。
- 右侧面板改为竖向模块导航：短语切分、状态调整、节奏控制。
- 底部预览复用提词窗口组件的布局，而不是另写简化版。

### 2.4 切分面板：像逐字调试，不像节奏调整台

当前问题：

- 当前句被拆成逐字和小圆点，视觉上非常工程化。
- 切分点很难理解，也没有“短语之间插入 / 取消”的语义。
- 缺少停顿设置、强调设置、当前句预览。
- 当前句字号不够大，用户无法在面板中确认语气。

目标状态：

- 切分面板应接近 `sotto-segmentation-panel-v1.png`：大号当前句、横向短语卡片、发光竖向切分点、停顿 / 强调分段控件、当前句预计耗时和底部预览。

V2 改法：

- 不再显示每个字符的小圆点。
- 显示短语卡片行，卡片之间显示发光切分点。
- 点击短语间的切分点用于取消；点击短语内部的可选字间位置时进入“插入点选择”轻状态。
- 加入停顿分段控件：无、短停顿、长停顿。
- 加入强调分段控件：普通、强调。
- 面板底部嵌入当前句提词预览。

### 2.5 提词浮窗：功能可用，但不像核心产品

当前问题：

- 当前窗口是大号半透明卡片，缺少参考图中的横向提词屏比例和左侧状态栏。
- `SOTTO PROMPT`、`REC`、进度、波形线和控制栏没有形成控制台结构。
- 当前句是大号文本，但缺少中部横向聚光光束。
- 当前短语用引号表示，不是柔光扫读。
- 控制按钮可用，但视觉不像提词设备的一部分。

目标状态：

- 提词窗口是 Sotto 的核心体验，应最接近 `sotto-prompt-window-v1.png`。
- 左侧固定状态栏：`SOTTO PROMPT`、状态点、`REC`、时间、进度。
- 右侧阅读区：上一句弱化、当前句最大最亮、下 3 句逐层变弱。
- 中部横向柔光扫过当前短语。
- 底部：波形 / 节奏线 + 上一句 / 暂停 / 下一句 / 速度 - / 速度值 / 速度 + / EXIT。

V2 改法：

- 将 `TeleprompterWindowView` 重构为 `PromptCardLayout`，分为 sidebar、readingArea、rhythmBar、controls。
- 当前短语不再用引号表示，改为 `PhraseHighlightOverlay`：柔光矩形或渐变光束叠在当前短语附近。
- 引入 `PromptProgress`：当前句序号 / 总句数。
- 控制栏默认弱化，鼠标 hover 或窗口聚焦时增强。

### 2.6 视觉系统：现在只有暗色，没有 Sotto 质感

当前问题：

- 背景点阵过于均匀，缺少舞台光、空气颗粒和光雾层次。
- 卡片材质都是同一种半透明面板，层级单一。
- 字体没有区分品牌字、状态字、正文和控制台数字。
- 暖白光感不足，很多元素只是灰色。

目标状态：

- 建立 Sotto 专用 SwiftUI 视觉组件，而不是散落在页面里的颜色和面板修饰符。

V2 改法：

- 新增 `SottoStageBackground`：点阵 + 顶部舞台光 + 轻微光尘。
- 新增 `SottoGlassPanel`：按用途分为 `hero / workbench / prompt / utility` 四档。
- 新增 `SottoStatusBadge`：`READY / PREPARING / ON AIR / PAUSED`。
- 新增 `SottoRhythmLine`：首页、准备中、预览和提词窗口复用。
- 新增 `SottoIconButton`：控制栏按钮统一尺寸、hover 和弱化规则。

## 3. V2 改造优先级

### P0：必须先改

1. 提词浮窗重构为正式 Sotto Prompt 结构。
2. 切分面板从逐字圆点改成短语卡片 + 发光切分点。
3. 编辑页从普通大文本框改成稿件准备台布局。
4. 首页从左右工具布局改成纵向演出后台入口。

### P1：增强体验

1. 加入准备中状态。
2. 加入自动保存状态和最近稿件状态点。
3. 当前短语改为柔光扫读，不再用引号。
4. 控制栏 hover / 弱化 / 浮现。

### P2：后续 polish

1. 点阵呼吸和舞台光细节。
2. 页面来源连续转场。
3. 更精致的品牌字标和状态字标。
4. 可调字号、透明度、窗口位置设置浮层。

## 4. 建议实施顺序

### 阶段一：建立视觉组件底座

- 抽出 Sotto 颜色、字体、间距、阴影、玻璃面板和状态组件。
- 重写背景层，让所有页面共享舞台空间。
- 不改业务逻辑，不动测试。

### 阶段二：先重做提词浮窗

- 因为它是核心产品体验，也是用户最容易判断“像不像 Sotto”的界面。
- 完成后先单独验收浮窗，再继续改主窗口。

### 阶段三：重做切分面板和底部预览

- 切分面板直接影响“节奏调整台”的感觉。
- 底部预览应复用提词浮窗组件，避免两套视觉。

### 阶段四：重做首页和编辑页

- 首页负责第一印象。
- 编辑页负责长时间工作体验。
- 这两页可以在视觉组件底座稳定后整体改。

## 5. 验收标准

V2 不能只看功能是否还在，而要看这些体验是否成立：

- 打开首页时，第一感觉是“演出后台”，不是“文本输入工具”。
- 点击准备上场后，能感觉系统在“整理节奏”，不是瞬间跳页。
- 编辑页里，稿件是主角，右侧切分和底部预览都围绕稿件服务。
- 切分面板让用户理解“我在调整表达节奏”，不是“我在调字符索引”。
- 提词浮窗第一眼像一个安静的舞台提词屏，而不是普通弹窗。
- 当前句、未来句、控制栏和状态栏的层级一眼清楚。
- 当前短语是被柔光照亮，不是被符号包起来。

## 6. 对当前代码的处理建议

保留：

- SwiftPM 工程结构。
- SottoCore 模型、切分、会话、保存和现有测试。
- `script/build_and_run.sh` 与 Codex Run 配置。
- AppKit 浮窗控制器的窗口创建能力。

重构：

- `HomeView`
- `EditorView`
- `SegmentPanelView`
- `SpotlightPreviewView`
- `TeleprompterWindowView`
- `SottoStyle`

新增：

- `SottoStageBackground`
- `SottoGlassPanel`
- `SottoStatusBadge`
- `SottoRhythmLine`
- `PromptCardLayout`
- `PhraseHighlightOverlay`
- `PreparingView`

避免：

- 不要在现有页面上继续叠控件。
- 不要用更多按钮和 chip 修补体验。
- 不要把参考图理解成“暗色 + 圆角 + 点阵”的表层组合。
- 不要在 V2 中引入 AI、语音识别、WebGL 或 Ghost Mode，先把本地体验做准。
