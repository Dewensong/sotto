## 交互含义

这张图用于承接用户粘贴稿件后的短暂等待过程：

1. 系统正在识别段落结构。
2. 系统正在切分句子与自然停顿。
3. 系统正在生成短语高亮节奏。
4. 完成后进入稿件编辑页。

该状态的作用不是展示技术进度，而是保持「演出后台」的仪式感，让等待过程也像准备上场的一部分。

## 设计原则

- 等待状态要安静、克制，不像系统 loading。
- 不使用机械进度条或强警告色。
- 状态步骤要清楚，但不要太技术化。
- 点阵声波和舞台光可以强化 Sotto 的产品气质。
- 处理中状态应让用户觉得产品正在认真整理稿件，而不是卡住。
