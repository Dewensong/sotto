## 2. 核心气质

Sotto 的 UI 气质是：

> 一个安静、精致、有仪式感的演出后台。

它不是普通提词器，不是文本编辑器，不是剪辑软件，也不是项目管理器。它应该让用户感觉：

- 我正在进入一个准备表达的空间。
- 我的稿件正在被安静地整理成节奏。
- 我可以慢慢把真正想说的话说清楚。
- 产品在后台陪我准备，而不是催我完成任务。

关键词：

- 暗色。
- 微光。
- 点阵。
- 舞台光。
- 呼吸感。
- 聚光。
- 后台。
- 低干扰。
- 安心。

## 3. 品牌与命名

产品名：

- `Sotto`

气质句：

- `a quiet backstage for speaking well`

品牌含义：

- `Sotto` 来自低声、轻声表达的联想。
- 它强调安静陪伴、低干扰提示、录制前的后台感。
- 对外可以写作 `Sotto by Ong`，但当前 MVP UI 里优先突出 `Sotto`。

品牌出现方式：

- 首页使用较大的 `Sotto` 标识。
- 功能页面可以使用右上角或侧栏的小型 `SOTTO`。
- 正式提词窗口使用 `SOTTO PROMPT` 作为状态标识。

产品名动效：

- `Sotto` 标识可以参考 React Bits `FuzzyText` 的轻微模糊 / 信号漂移效果。
- 该效果只用于品牌字、状态字标或上场前的仪式瞬间，不用于正文稿件。
- 默认应接近点阵微微失焦，而不是故障风格。
- hover 时可以略微清晰或轻微漂移，但不能影响产品名识别。
- 不使用强 glitch、点击爆发、彩色渐变或高频抖动。

## 4. 视觉语言

### 背景

默认使用暗色背景。

核心背景元素：

- 黑色或近黑色底。
- 低亮度点阵网格。
- 低亮度像素颗粒幕布。
- 轻微颗粒感。
- 局部舞台光束。
- 细微星点 / 光尘。

背景不应该：

- 变成纯色扁平黑。
- 使用大面积彩色渐变。
- 使用明亮、拥挤或商业 SaaS 风背景。

### 光感

Sotto 的光不是装饰，而是状态与注意力。

主要光感来源：

- 顶部舞台光束。
- 卡片边缘柔光。
- 当前句 / 当前短语聚光。
- 状态点微光。
- 点阵声波和节奏线。

光感原则：

- 光要有方向感，像舞台或控制台。
- 光要克制，不要过曝。
- 光要服务当前内容，不要抢正文。
- 首页顶部舞台光可以参考 React Bits `LightRays` 的光束生成方式，但应使用暖白 / 低饱和、慢速、低噪声版本。
- 鼠标影响如果开启，只能非常轻，像空气扰动，不应像光束追着鼠标跑。

### 点阵

点阵是 Sotto 的核心视觉识别之一。

使用位置：

- 背景网格。
- 产品标识。
- 状态字体。
- 提词窗口背景。
- 声波 / 节奏线。

点阵原则：

- 用来建立技术与舞台的混合气质。
- 不要让正文全部变成难读的点阵字。
- 正文提词可以带点阵质感，但可读性必须优先。
- 默认背景点阵可以参考 Magic UI `DotPattern`，用 SVG 实现低成本、稳定、可降级的点阵底座。
- 点阵背景不应完全静止，默认要有非常缓慢的呼吸感。
- `glow` 可以全局存在，但强度按场景分级：首页 / 准备中更明显，编辑区 / 提词窗口更弱、更慢。
- 点阵颜色跟随暖白 / 低亮灰，不单独变成彩色装饰层。

### 像素颗粒幕布

像素颗粒幕布可以作为星光点阵的替代或补充，参考 React Bits `PixelBlast` 的背景思路。

适用位置：

- 首页背景。
- 自动切分处理中状态。
- 空状态 / 等待状态。
- 关键上场转场的短暂背景层。

原则：

- 它应该像暗场里的像素空气，不像游戏粒子特效。
- 优先使用圆形或细小方形像素，颜色保持暖白、低亮灰或低饱和淡青。
- 像素密度要留出呼吸空间，不要铺满整个画面。
- 可以替代部分星光点阵，但不应让产品失去 Sotto 的点阵识别。
- 提词窗口正文背后只允许极弱版本，必要时直接关闭。

不应该：

- 做成高密度粒子爆炸。
- 使用强 ripple / liquid 鼠标互动。
- 使用高饱和紫色或霓虹色作为默认背景。
- 让像素运动比文字、卡片和顶部舞台光更显眼。

### 外部 WebGL 场景

外部 WebGL 场景可以作为首页和准备状态的沉浸式背景候选，例如 Unicorn Studio 项目 `xbdYsuMp0mSR01MXofEj`。

原则：

- 只作为背景氛围层，不承载核心信息。
- 必须有明确宽高，不能依赖内容自动撑开。
- 默认懒加载，只在可见区域运行。
- 颜色、速度和运动密度必须服从 Sotto 的暗色、微光、低干扰原则。
- 提词窗口和正文编辑区不默认使用外部 WebGL 场景。
- 必须准备静态降级背景，避免 WebGL 或网络加载失败时出现空白。

### 未来明暗模式切换

明亮 / 黑暗模式切换可以参考 Magic UI `AnimatedThemeToggler` 的思路：用 View Transitions API 做一次从按钮或视口中心展开的柔和换光。

本期不实现主题切换，只记录为后续能力储备。

后续原则：

- Sotto 默认仍以暗色模式为主，明亮模式是后续扩展，不反向影响本期暗色基准。
- 主题切换应像舞台换光，不像页面重载。
- 默认优先使用 `circle` 或柔和 `rectangle` reveal，不使用星形、三角形等过强的装饰形状。
- 转场起点可以来自主题按钮；如果未来做全局设置面板，也可以从视口中心展开。
- 转场只用于明暗主题切换，不复用到普通页面跳转、弹窗和提词状态变化。
- 不支持 View Transitions API 时，应直接切换主题，不阻塞主要操作。

## 5. 布局原则

### 页面主次

每个页面必须只有一个主角：

- 首页：材料入口和 `ON STAGE` 主按钮。
- 处理中状态：`正在整理这段稿子的提词节奏`。
- 稿件编辑页：主稿件文本。
- 切分面板：当前选中句子和切分点。
- 正式提词窗口：当前句。

辅助内容必须围绕主角服务，不要抢焦点。

### 空间关系

整体布局偏「舞台中央 + 后台控制」：

- 重要内容居中或偏中。
- 辅助控制放侧边或底部。
- 状态信息放顶部或左侧。
- 控制按钮尽量收敛成一条控制栏。

不要使用复杂多栏后台、密集表格或大量工具栏。

### 卡片

主要卡片形态：

- 暗色半透明。
- 大圆角。
- 细边框。
- 轻微发光。
- 内部有点阵或低亮度纹理。
- 可带轻微反光玻璃质感，像舞台后台的黑色玻璃面板。

卡片用途：

- 材料入口。
- 准备状态容器。
- 提词窗口。
- 当前句预览。
- 切分操作区。

卡片不能堆太多层，避免变成卡片套卡片的后台界面。

### 滚动边缘渐进模糊

滚动边缘渐进模糊可以参考 Magic UI `ProgressiveBlur`：在可滚动内容的顶部或底部叠加多层轻微 blur，柔和提示内容仍在延续。

适用位置：

- 稿件编辑区顶部 / 底部。
- 最近稿件列表。
- 切分面板的长内容区域。
- 底部提词预览区。
- 正式提词窗口的上一句 / 未来句边缘弱化。

原则：

- 它是阅读保护层，不是氛围装饰。
- 模糊应从清晰内容自然过渡到边缘暗化。
- 不遮挡当前句、当前短语和关键按钮。
- 容器必须是 `pointer-events: none`，不能影响选中文本或点击。
- 提词窗口里只允许极轻版本，优先保护当前句可读性。

### 反光玻璃卡片

反光玻璃卡片可以参考 React Bits `ReflectiveCard` 的材质思路，但不照搬其摄像头、身份卡和安防视觉。

适用位置：

- 首页材料入口卡片。
- 自动切分处理中容器。
- 底部提词预览卡片。
- 正式提词窗口外壳。
- 当前句 / 切分操作区的选中态。

原则：

- 反光只作为边缘、斜向高光和轻微材质噪声存在。
- 卡片应像暗色玻璃，不像金属证件或银行卡。
- 不默认调用摄像头，不使用真实视频作为卡片背景。
- 不使用 `SECURE ACCESS`、锁、指纹等安防语义。
- 提词窗口里的反光必须弱于正文聚光。

## 6. 字体与排版

### 字体气质

Sotto 可以使用两类字体气质：

1. **点阵 / 像素字体**：用于品牌、状态、短标签、控制台元素。
2. **高可读正文字体**：用于中文稿件、提词正文、说明文字。

原则：

- 提词正文必须优先可读。
- 大号当前句可以有点阵质感，但不能牺牲识别速度。
- 小号说明文字不要过度像素化。

### 中文排版

中文口播稿需要：

- 足够大的字号。
- 宽松行距。
- 明确段落。
- 当前内容高亮。
- 未来内容可读但弱化。

避免：

- 字太小。
- 字距过散导致读起来断裂。
- 装饰性字体影响中文识别。

## 7. 颜色规则

### 主色

主色是近黑、暖白和低亮度灰。

推荐关系：

- 背景：近黑。
- 主文字：暖白。
- 次级文字：低亮灰。
- 边框：半透明暖白 / 灰白。
- 高亮：暖白光、淡金、淡青绿。

### 状态色

状态色要少。

- 绿色：`READY`、在线、可继续。
- 白色 / 暖白：`PREPARING`、当前聚光。
- 黄色：待续、需要注意。
- 蓝色：已切分、信息状态。

状态色只用于点、短标签和轻量提示，不大面积铺开。

### 禁忌

- 不使用大面积高饱和颜色。
- 不使用彩虹式状态系统。
- 不使用红色警告感，除非真的是错误。
- 不让颜色比文字更抢眼。

## 8. 核心组件

### 状态点

状态点用于表达系统正在陪伴用户。

常见状态：

- `READY.`
- `PREPARING`
- `在线`
- `REC`

状态点应带轻微发光，但不闪烁得过于明显。

### 主 CTA

首页主 CTA 是仪式按钮。

当前参考：

- `ON STAGE`
- `准备上场`

规则：

- 主 CTA 要像上场按钮，而不是普通提交按钮。
- 可以有胶囊形、发光边缘、点阵质感。
- 文案要短。
- 点击确认可以参考 React Bits `ClickSpark` 的短促火花，但必须很轻，只表达「上场确认」。

### 确认火花

确认火花用于极少数高意义点击，不是全局点击反馈。

适用位置：

- 首页 `ON STAGE`。
- 切分点插入成功。
- 准备步骤完成切换。
- 提词窗口开始 / 继续的轻确认。

原则：

- 使用暖白、淡金或低饱和青绿。
- 火花数量少、半径小、持续时间短。
- 火花应像微光散开，不像庆祝礼花。
- 不包裹整页，不让每次点击都有火花。

### 控制栏

正式提词窗口和预览区的控制栏包含：

- 上一句。
- 暂停 / 继续。
- 下一句。
- 速度降低。
- 当前速度。
- 速度提高。
- 退出。

规则：

- 控制栏低调。
- 图标优先。
- 分隔线可以细而清楚。
- 控制不要抢当前句。

### 分段控件

用于停顿和强调。

示例：

- 无 / 短停顿 / 长停顿。
- 普通 / 强调。

规则：

- 分段控件可以有柔光选中态。
- 选中态要明确。
- 不要像普通网页 tab。

### 切分点

切分点是切分面板的核心交互元素。

视觉要求：

- 有明确触点。
- 可发光。
- 可以用竖线或节点表达。
- 点击插入 / 取消切分点。

切分点要像节奏标记，不像代码分隔符。

## 9. 页面级规则

### 首页

保留主线：

- `Sotto`
- `a quiet backstage for speaking well`
- `今天想让哪段想法上场？`
- `ON STAGE / 准备上场`

首页要有舞台光、点阵、材料入口和最近稿件。

### 处理中状态

保留主线：

- `PREPARING`
- `正在整理这段稿子的提词节奏`
- 三步流程：识别段落、切分句子、生成短语高亮节奏。
- 底部点阵声波。

处理中状态要像准备上场，不像系统 loading。

### 稿件编辑页

保留主线：

- 左侧主稿件。
- 右侧辅助面板。
- 底部提词预览。
- 顶部稿件标题和自动保存状态。
- `打开提词窗口` 入口。

稿件编辑页不要变成复杂字幕软件。

### 切分面板

保留主线：

- 大号当前句。
- 横向短语切分卡片。
- 发光切分点。
- 停顿 / 强调设置。
- 当前句耗时。
- 底部当前句预览。

切分面板要像节奏调整台。

### 正式提词窗口

保留主线：

- 横向悬浮卡片。
- 左侧 `SOTTO PROMPT` 状态栏。
- 当前句最大最亮。
- 上一句弱化。
- 未来句可读。
- 底部节奏线和控制栏。

正式提词窗口是 Sotto 的核心体验，所有视觉都要服务当前句。

## 10. 动效与状态

### 动效方向

动效应该慢、稳、轻。

适合的动效：

- 呼吸光。
- 柔光扫过当前短语。
- 状态点轻微闪烁。
- 点阵声波轻微波动。
- 卡片淡入。
- 页面转场轻微推进。

不适合的动效：

- 弹跳。
- 大幅位移。
- 强烈闪烁。
- 游戏化反馈。
- 过快过密的 loading 动画。

### 高亮方式

当前短语高亮采用：

> 柔光扫读 + 文字微亮。

不要使用传统 KTV 的强色块填字。

## 11. 文案语气

Sotto 的文案要像安静的后台工作人员，不像客服、老师或工具提示。

文案原则：

- 短。
- 稳。
- 有一点仪式感。
- 不催促。
- 不制造失败感。

推荐语气：

- `今天想让哪段想法上场？`
- `正在整理这段稿子的提词节奏`
- `舞台尚未点亮，但每一次呼吸都被听见`
- `专注表达，聚光成句。`

避免：

- `请上传文件以开始处理`
- `正在执行任务`
- `错误：解析失败`
- `请立即修正`

## 12. 实现注意事项

### 可读性优先

所有视觉效果都不能牺牲提词可读性。特别是：

- 当前句必须足够清楚。
- 下 3 句必须能提前读到。
- 控制栏不能遮挡正文。
- 点阵和发光不能让中文模糊。

### MVP 0.1 范围

当前 UI 设计基准只覆盖 MVP 0.1：

- 粘贴已有稿件。
- 自动切分。
- 手动切分。
- 节奏设置。
- 提词预览。
- Mac 本机提词窗口。

不覆盖：

- AI 生成稿件。
- 要点模式。
- 语音识别。
- 第二屏。
- Ghost Mode。
- 创作蓝图。

### 设计可变性

当前参考图是方向，不是约束死的最终稿。后续可以调整：

- 页面布局。
- 组件形态。
- 文案细节。
- 图标。
- 具体颜色。
- 动效强度。

但调整时应尽量保留：

- 演出后台隐喻。
- 暗色点阵语言。
- 聚光当前句。
- 安静、安心、低干扰。

## 13. 给设计 AI / 实现 AI 的提示
